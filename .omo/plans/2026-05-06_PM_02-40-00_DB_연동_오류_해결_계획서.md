# 2026-05-06_PM_02-40-00_DB_연동_오류_해결_계획서

## 1. 현상 분석
- **오류 발생**: `GET /api/statistics/subjective-count` API 호출 시 500 에러 발생
- **로그 내용**: `org.postgresql.util.PSQLException: ERROR: relation "subjective_problems" does not exist`
- **문제점**: 데이터베이스에 `subjective_problems` 테이블이 존재하지 않아 쿼리 실행 실패

## 2. 원인 파악
- **설정 확인**: `backend/src/main/resources/application.properties` 파일에 `spring.jpa.hibernate.ddl-auto=none`으로 설정되어 있어 엔티티가 테이블로 자동 생성되지 않음
- **누락 확인**: `subjective_problems` 및 `programming_language_problems` 테이블이 스키마 설계에는 포함되었으나, 실제 Supabase PostgreSQL 서버에는 반영되지 않음
- **초기화 파일 부재**: `backend/src/main/resources/db/schema.sql` 파일이 비어 있어 수동/자동 테이블 생성이 이루어지지 않음

## 3. 해결 전략 (Fix Plan)

### Step 1: 테이블 생성 SQL 스크립트 작성
누락된 `subjective_problems` 및 `programming_language_problems` 테이블을 생성하는 SQL 문을 준비합니다.

```sql
-- 1. subjective_problems 테이블 생성
CREATE TABLE IF NOT EXISTS "subjective_problems" (
    "id" BIGSERIAL PRIMARY KEY,
    "subject_id" BIGINT,
    "question" VARCHAR(1000) NOT NULL,
    "answer" VARCHAR(500) NOT NULL,
    "explanation" VARCHAR(2000),
    "difficulty" INT,
    "is_ai_generated" BOOLEAN NOT NULL DEFAULT FALSE,
    "created_at" TIMESTAMP WITHOUT TIME ZONE,
    "updated_at" TIMESTAMP WITHOUT TIME ZONE,
    CONSTRAINT "fk_subjective_problems_subject" FOREIGN KEY ("subject_id") REFERENCES "subject"("id") ON DELETE CASCADE
);

-- 2. programming_language_problems 테이블 생성
CREATE TABLE IF NOT EXISTS "programming_language_problems" (
    "id" BIGSERIAL PRIMARY KEY,
    "subject_id" BIGINT,
    "prog_language" VARCHAR(64) NOT NULL,
    "question" VARCHAR(1000) NOT NULL,
    "answer" VARCHAR(500) NOT NULL,
    "explanation" VARCHAR(2000),
    "difficulty" INT,
    "is_ai_generated" BOOLEAN NOT NULL DEFAULT FALSE,
    "created_at" TIMESTAMP WITHOUT TIME ZONE,
    "updated_at" TIMESTAMP WITHOUT TIME ZONE,
    CONSTRAINT "fk_prog_problems_subject" FOREIGN KEY ("subject_id") REFERENCES "subject"("id") ON DELETE CASCADE
);

-- 3. 인덱스 생성
CREATE INDEX IF NOT EXISTS "idx_subjective_problems_subject_id" ON "subjective_problems"("subject_id");
CREATE INDEX IF NOT EXISTS "idx_prog_problems_subject_id" ON "programming_language_problems"("subject_id");
```

### Step 2: DB 반영 및 초기화 설정 변경
1. `backend/src/main/resources/db/schema.sql`에 위 SQL 내용을 기록합니다.
2. (선택 사항) 로컬 테스트를 위해 `application.properties`에서 `spring.jpa.hibernate.ddl-auto=update`로 일시 변경하여 테이블 생성을 유도할 수도 있으나, Supabase 권한 및 정합성을 위해 SQL 직접 실행을 권장합니다.

### Step 3: 검증 (Validation)
1. 백엔드 서버를 재시작합니다.
2. `GET /api/statistics/subjective-count` API를 다시 호출하여 오류가 사라지는지 확인합니다.
3. 데이터베이스 툴(Supabase Dashboard 등)을 통해 테이블 구조가 엔티티와 일치하는지 최종 확인합니다.
