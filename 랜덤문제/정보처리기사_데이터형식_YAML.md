# 정보처리기사 데이터 형식 - YAML 주관식 단답형 문제

> **정답 키워드:** YAML
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 질접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 사람이 쉽게 읽고 쓸 수 있도록 설계된 데이터 직렬화 형식으로, 들여쓰기를 사용하여 계층 구조를 표현하는 것이 특징인 데이터 형식을 **[     ]** 이라고 한다. 이 형식은 주로 설정 파일과 데이터 교환에 사용된다.

**정답:** YAML

---

## 문제 2 (직접 질문 유형)

**질문:** XML과 JSON보다 사람이 읽고 쓰기 쉬운 데이터 직렬화 형식으로, 들여쓰기와 줄바꿈을 사용하여 계층 구조를 표현하며, 주로 Docker Compose, Kubernetes, Ansible 등의 설정 파일에 사용되는 데이터 형식의 이름은 무엇인가?

**정답:** YAML

---

## 문제 3 (빈칸 채우기 + 비교 설명 유형)

**질문:** 데이터 직렬화 형식 중 **[     ]** 는 XML의 복잡한 태그 구조와 JSON의 중괄호 구조보다 사람이 읽고 쓰기 쉬운 문법을 제공한다. 이 형식은 주석을 지원하며, 스칼라, 리스트, 딕셔너리 등 다양한 데이터 타입을 표현할 수 있다. Docker Compose 파일과 Kubernetes 매니페스트 파일에서 주로 사용된다.

**정답:** YAML

---

## AI 프롬프트 작성 가이드

YAML를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: YAML
- 유형: 빈칸 채우기와 직접 질문을 혼합
- YAML의 결정적 키워드(데이터 직렬화, 사람이 읽기 쉬움, 들여쓰기, 계층 구조, 설정 파일, Docker Compose, Kubernetes)를 문제 설명에 배치하여 정답을 유도
- XML, JSON과 비교하여 YAML의 특징을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: YAML"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "들여쓰기와 줄바꿈" vs "태그 구조" (XML), "중괄호 구조" (JSON), "주석 지원" vs "주석 미지원" (JSON) 등 각 데이터 형식의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
