# 정보처리기사 암호화 - DES 문제

## AI 프롬프트 작성 가이드

- **정답 키워드 명시**: 프롬프트에 "정답: DES"를 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
- **유형 지정**: 1.빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제만 랜덤형식의 질문으로 다양성이 보장되어 작성한다. 2.정답은 (DES)으로 고정된다.
- **결정적 구분자 제시**: (DES)에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.

---

## 문제 1 (빈칸 채우기)

1977년 미국 국가표준기술연구소(NIST)에서 공표한 대칭키 블록 암호화 알고리즘으로, 64비트 블록 크기와 56비트 키 길이를 사용하며 16라운드의 파이스텔(Feistel) 구조를 적용하는 알고리즘은 (DES)이다.

**정답:** (DES)

---

## 문제 2 (직접 질문)

대칭키 암호화 방식 중 하나로, 64비트 평문 블록을 56비트 비밀키로 16번의 라운드 연산을 거쳐 암호문으로 변환하며, 현재는 키 길이가 짧아 보안성이 취약해져 3DES나 AES로 대체된 전통적인 암호 알고리즘의 이름은 무엇인가?

**정답:** (DES)

---

## 문제 3 (비교 설명)

다음 중 1970년대 개발되어 오랫동안 대표적인 대칭키 암호 표준으로 사용되었으나, 키 길이가 56비트로 제한되어 현재는 보안 취약성으로 인해 권장되지 않는 초기 블록 암호화 알고리즘은 무엇인가?

**정답:** (DES)
