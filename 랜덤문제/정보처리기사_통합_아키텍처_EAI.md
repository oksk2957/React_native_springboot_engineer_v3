# 정보처리기사 통합 아키텍처 - EAI 주관식 단답형 문제

> **정답 키워드:** EAI
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 기업 내 다양한 애플리케이션 시스템 간의 데이터 교환과 통합을 지원하는 중간 소프트웨어를 **[     ]** 라고 한다. 이는 기업 내 서로 다른 플랫폼과 애플리케이션 간의 데이터 연동과 통합을 가능하게 한다.

**정답:** EAI

---

## 문제 2 (직접 질문 유형)

**질문:** 기업 내 서로 다른 애플리케이션 시스템 간의 데이터 교환과 통합을 지원하는 기술로, 메시지 기반 통신, 데이터 변환, 프로세스 통합 등을 제공하는 엔터프라이즈 애플리케이션 통합의 약자는 무엇인가?

**정답:** EAI

---

## 문제 3 (빈칸 채우기 + 비교 설명 유형)

**질문:** 기업 내 다양한 시스템 간의 데이터 통합을 지원하는 **[     ]** 는 포인트 투 포인트(Point-to-Point) 방식의 한계를 극복하고, 중앙 집중식 통합 허브를 통해 시스템 간의 느슨한 결합을 가능하게 한다. 이는 기업의 비즈니스 프로세스 통합과 실시간 데이터 동기화를 지원한다.

**정답:** EAI

---

## AI 프롬프트 작성 가이드

EAI를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: EAI
- 유형: 빈칸 채우기와 직접 질문을 혼합
- EAI의 결정적 키워드(엔터프라이즈 애플리케이션 통합, 기업 내 시스템 통합, 데이터 교환, 메시지 기반 통신, 포인트 투 포인트와의 비교)를 문제 설명에 배치하여 정답을 유도
- 포인트 투 포인트 방식과 비교하여 EAI의 장점을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: EAI"를 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "엔터프라이즈 애플리케이션 통합" vs "단순 데이터 연동", "중앙 집중식 통합 허브" vs "포인트 투 포인트" 등 각 기술의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
