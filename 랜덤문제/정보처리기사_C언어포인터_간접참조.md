# 정보처리기사 C언어 포인터 간접 참조 - * 연산자의 값 읽기 + 값 쓰기 주관식 단답형 문제

> **정답 키워드:** * (간접 참조 연산자)
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** C언어에서 포인터 변수 `p`가 변수 `a`의 주소를 저장하고 있을 때(`int *p = &a;`), 포인터 `p` 자체는 주소만 가리키므로 값을 직접 조작할 수 없다. 그러나 `*p`와 같이 **[     ]** 연산자를 붙이면 주소를 따라가 실제 메모리 위치의 값에 접근할 수 있다. 즉, `*p`를 통해 해당 주소의 값에 대해 **값 읽기**(`printf("%d", *p)`)와 **값 쓰기**(`*p = 100`)가 모두 가능하다.

**정답:** *

**개발 생각:** 빈칸 채우기 유형은 정보처리기사 실기 시험에서 가장 빈출되는 출제 형식이다. 이 문제는 포인터 `p`(주소)와 `*p`(값 접근)의 차이를 명확히 이해하는 것이 핵심이며, `*` 연산자가 "간접 참조(dereference)"를 수행하여 값 읽기와 값 쓰기를 모두 가능하게 한다는 점이 정답의 핵심 구분자이다. 프롬프트 작성 시 "포인터에 붙여 값 접근을 가능하게 하는 연산자"를 묻는 빈칸 문제, 정답은 `*`라고 지정하면 AI가 정확히 유도한다.

---

## 문제 2 (직접 질문 유형)

**질문:** C언어에서 포인터가 가리키는 주소의 실제 데이터에 접근하기 위해 사용하는 연산자는 무엇이며, 이 연산자를 통해 수행할 수 있는 두 가지 작업을 설명하시오. 포인터 변수는 메모리 주소만 저장하므로 `p`만으로는 주소를 출력할 수 있지만 값을 직접 조작하지 못한다. 그러나 해당 연산자를 사용하면 주소를 따라가 해당 메모리 위치의 값을 확인(`printf("%d", *p)`)하고 변경(`*p = 100`)할 수 있다.

**정답:** * 연산자(간접 참조 연산자), 값 읽기 + 값 쓰기

**개발 생각:** 직접 질문 유형은 간접 참조 연산자의 이름과 기능을 직접 묻는 형식으로, 정보처리기사 실기에서 "연산자의 의미와 기능을 설명하시오"라는 출제 방식에 대응한다. 정답은 두 부분으로 구성된다: (1) 연산자 이름 = `*`(간접 참조 연산자, dereference operator), (2) 가능한 작업 = 값 읽기 + 값 쓰기. "포인터는 주소만 저장" vs "간접 참조는 값에 접근"이라는 대비가 결정적 구분자이다. 많은 학습자가 포인터의 간접 참조를 "값 읽기만 가능"으로 오해하는 경우가 많으므로, "값 쓰기도 가능"이라는 점이 오답과 정답을 구분하는 핵심이다.

## 문제 3 (비교 설명 유형)

**질문:** C언어에서 포인터 변수 `p`와 `*p`의 차이를 비교 설명하시오. 포인터 `p`는 변수의 **주소**만 저장하므로 `printf("%p", p)`로 주소를 출력할 수 있지만 값을 직접 조작하지 못한다. 반면 `*p`는 **[     ]** 연산자를 통해 주소를 따라가 실제 값에 접근하며, 이를 통해 **값 읽기**와 **값 쓰기**가 모두 가능하다. 즉, `p`는 "주소만 가리키는 역할"이고 `*p`는 "주소를 따라가 값을 조작하는 역할"이다. Java에서는 참조 변수가 이 두 역할을 자동으로 수행하여 `*` 연산자가 필요 없고 `.`(dot) 연산자로 바로 접근한다.

**정답:** *

**개발 생각:** 비교 설명 유형은 포인터 `p`(주소)와 `*p`(값 접근)의 역할 차이를 동시에 제시하여, 두 개념의 대응 관계를 명확히 구분한다. 이 유형은 정보처리기사 시험에서 "포인터와 간접 참조의 차이점을 비교 설명"하는 변형으로 자주 출제된다. 결정적 구분자는 "p = 주소만 가리킴(값 직접 조작 불가)" vs "*p = 주소를 따라가 값에 접근(값 읽기 + 값 쓰기 모두 가능)"의 대비이다. Java 비교를 포함한 것은 "Java 참조 = C 포인터 + * 연산자의 안전한 버전"이라는 확장 개념을 제공하여, 정보처리기사에서 C와 Java의 포인터/참조 차이를 묻는 문제에도 대응한다.

## AI 프롬프트 작성 가이드

C언어 포인터 간접 참조(* 연산자)의 "값 읽기 + 값 쓰기"를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: * (간접 참조 연산자)
- 정답 설명: 포인터에 * 연산자를 붙여 간접 참조 시 값 읽기 + 값 쓰기가 모두 가능
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- 각 문제에 정답과 개발 생각(해설)을 포함
- 포인터 간접 참조의 결정적 키워드(* 연산자, 간접 참조, dereference, 주소 따라가기, 값 읽기, 값 쓰기)를 문제 설명에 배치하여 정답을 유도
- p(주소만 가리킴)와 *p(값에 접근)의 비교 설명을 1개 문제에 포함
- "값 읽기만 가능"이라는 오답과 "값 읽기 + 값 쓰기 모두 가능"이라는 정답을 구분하는 결정적 구분자를 명시
- Java 참조와 C 포인터의 차이를 비교 설명 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: *"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "포인터 p = 주소만 가리킴(값 직접 조작 불가)" vs "간접 참조 *p = 주소를 따라가 값에 접근(값 읽기 + 값 쓰기 모두 가능)"이라는 역할 대비를 프롬프트에 포함하면, AI가 포인터와 간접 참조를 명확히 구분하는 문제를 생성한다. "값 읽기만 가능"이라는 오답 유형과 "값 읽기 + 값 쓰기" 정답의 구분이 핵심 구분자이다.
4. **개발 생각 요청**: "개발 생각(해설)을 포함"이라고 지정하면, AI가 문제 설계 의도와 출제 패턴 분석을 함께 제공하여 학습 효율이 향상된다.
