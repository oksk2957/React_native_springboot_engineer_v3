# 정보처리기사 보안 - SYN 플러드

## AI 프롬프트 작성 가이드

- **정답 키워드 명시**: 프롬프트에 "정답: ( )"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
- **유형 지정**: 1.빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제만 랜덤형식의 질문으로 다양성이 보장되어 작성한다. 2.정답은 ()으로 고정된다.
- **결정적 구분자 제시**: ( )에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
- **해설 미포함**: 질문과 정답만 작성하며 해설은 작성하지 않는다.

---

## 문제 1

**유형**: 빈칸 채우기

공격자가 TCP 3-way 핸드셰이크 과정에서 SYN 패킷만 대량으로 전송하고 SYN-ACK에 대한 응답을 보내지 않아 서버의 자원을 고갈시키는 공격 기법을 ( )이라고 한다.

**정답**: SYN 플러드

---

## 문제 2

**유형**: 직접 질문

SYN 플러드 공격으로 인해 서버의 연결 대기 큐(Backlog Queue)가 가득 차 정상적인 연결 요청을 처리하지 못하게 되는 상태를 무엇이라 하는가?

**정답**: 서비스 거부(DoS)

---

## 문제 3

**유형**: 비교 설명

SYN 플러드 공격을 방어하기 위해 SYN 패킷 수신 시 연결을 완전히 설정하지 않고 쿠키 형태의 정보를 생성하여 저장하는 기법을 무엇이라 하는가?

**정답**: SYN 쿠키
