# 정보처리기사 운영체제 - PCB 주관식 단답형 문제

> **정답 키워드:** PCB (Process Control Block)
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 운영체제에서 프로세스를 관리하기 위해 프로세스의 상태, 프로그램 카운터, 레지스터, 메모리 정보 등을 저장하는 자료구조를 **[     ]** 이라고 한다.

**정답:** PCB (Process Control Block)

---

## 문제 2 (직접 질문 유형)

**질문:** 운영체제가 프로세스의 식별자(PID), 프로세스 상태, 프로그램 카운터, CPU 레지스터, 메모리 관리 정보, 입출력 상태 정보 등을 저장하여 프로세스 스케줄링과 문맥 교환을 수행하는 데 사용하는 커널 자료구조의 이름은 무엇인가?

**정답:** PCB (Process Control Block)

---

## 문제 3 (비교 설명 유형)

**질문:** 운영체제에서 **프로세스**는 실행 중인 프로그램의 인스턴스로, 독립적인 메모리 공간을 가진다. 반면, 운영체제가 프로세스를 식별하고 관리하기 위해 유지하는 자료구조인 **[     ]** 은(는) 프로세스 식별자, 상태, 프로그램 카운터, 레지스터, 메모리 경계 정보 등을 저장하며, 문맥 교환 시 현재 실행 상태를 보존하고 복원하는 핵심 역할을 수행한다.

**정답:** PCB (Process Control Block)

---

## AI 프롬프트 작성 가이드

PCB를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: PCB (Process Control Block)
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- PCB의 결정적 키워드(프로세스 관리 자료구조, 프로세스 식별자 PID, 프로세스 상태, 프로그램 카운터, CPU 레지스터, 메모리 관리 정보, 문맥 교환, 커널 자료구조)를 문제 설명에 배치하여 정답을 유도
- 프로세스와 PCB의 관계를 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: PCB (Process Control Block)"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "프로세스 관리 자료구조" vs "프로세스 자체", "커널 자료구조" vs "사용자 공간", "문맥 교환 시 상태 보존" vs "실행 단위" 등 PCB의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
