# 정보처리기사 오류검출기법 - ARQ

## AI 프롬프트 작성 가이드

- **정답 키워드 명시**: 프롬프트에 "정답: ( )"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
- **유형 지정**: 1.빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제만 랜덤형식의 질문으로 다양성이 보장되어 작성한다. 2.정답은 ()으로 고정된다.
- **결정적 구분자 제시**: ( )에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
- **해설 미포함**: 질문과 정답만 작성하며 해설은 작성하지 않는다.

## 문제 1

**유형**: 빈칸 채우기

데이터 전송 중 오류가 발생하면 수신 측이 송신 측에 재전송을 요구하는 오류 제어 방식을 ( )이라고 한다.

**정답**: ARQ

Stop-and-Wait, Go-Back-N, Selective Repeat 등의 방식이 대표적으로 사용되는 오류 발생 시 재전송을 요구하는 오류 제어 기법의 영문 약어는 무엇인가?

**정답**: ARQ

## 문제 3

**유형**: 비교 설명

FEC는 오류를 스스로 정정하는 방식이고, ( )는 오류 발생 시 수신 측이 송신 측에 재전송을 요구하는 방식이다. 후자는 오류 검출 후 피드백을 통해 재전송을 수행하므로 전송 신뢰성이 높다.

**정답**: ARQ
