# 정보처리기사 - 대괄호 [] vs 중괄호 {} 비교 문제

> **유형**: 주관식 단답형  
> **관련 정답 키워드**: 크기(수량·위치), 값(내용·초기화리스트), 리스트, 셋  
> **결정적 구분자**: `[]` = 수량·위치, `{}` = 내용·값 (C/Java), `[]` = 리스트, `{}` = 셋 (Python)

---

## 문제 1

**[빈칸 채우기]**

C 언어에서 `int arr[5];` 선언 시 대괄호 `[5]`는 배열의 **( 1 )**을 지정하고,  
`int arr[] = {10, 20, 30};` 초기화 시 중괄호 `{10, 20, 30}`은 배열의 **( 2 )**을 지정한다.

- ( 1 ): 정답 → **크기** (또는 수량, 방 수)
- ( 2 ): 정답 → **값** (또는 내용, 초기화 리스트)

### 해설
- `[]`는 선언 위치에서 **"몇 개의 방을 만들 것인가"**를 결정 → 크기(수량)
- `{}`는 초기화 위치에서 **"방에 어떤 값을 넣을 것인가"**를 결정 → 값(내용)
- 개발자 생각: 배열은 아파트 건물과 같다. `[5]`는 "5층 건물을 지어라"이고, `{10,20,30}`은 "1층에 10, 2층에 20, 3층에 30을 배치하라"이다. []은 구조(틀), {}은 내용(채움)이라는 개발적 직관이 핵심이다.

## 문제 2

**[직접 질문]**

Python에서 `[1]`과 `{1}`은 각각 어떤 **데이터 타입**을 생성하는가?  
두 타입의 가장 큰 차이점 한 가지를 서술하라.

- `[1]`의 타입: 정답 → **리스트 (list)**
- `{1}`의 타입: 정답 → **셋 (set)**
- 주요 차이점: 정답 → **리스트는 순서가 있고 인덱스 접근이 가능하며 중복을 허용하고, 셋은 순서가 없고 인덱스 접근이 불가능하며 중복을 허용하지 않는다**

### 해설
- Python의 `[1]`은 리스트 리터럴: `type([1]) == list`, `a[0] == 1` (인덱스 접근 가능)
- Python의 `{1}`은 셋 리터럴: `type({1}) == set`, `b[0] → TypeError` (인덱스 접근 불가)
- ⚠️ 빈 `{}`는 **딕셔너리(dict)**이고, 빈 셋은 `set()`으로만 생성 가능 — 이것도 기출 출제 포인트
- 개발자 생각: C/Java에서 []과 {}는 같은 배열의 다른 속성(크기 vs 값)을 표현하지만, Python에서는 완전히 다른 컬렉션 타입을 결정하는 "타입 선택 기호"이다. 언어 설계자의 철학 차이 — C/Java는 배열 중심, Python은 컬렉션 다형성 중심.

## 문제 3

**[비교 설명]**

Java에서 `new int[1]`과 `new int[]{1}`은 배열의 `length`가 같지만  
`arr[0]`의 값이 다르다. 각각 `arr[0]`의 값을 쓰고,  
그 이유를 **초기화 관점**에서 설명하라.

- `new int[1]`의 `arr[0]`: 정답 → **0** (기본값)
- `new int[]{1}`의 `arr[0]`: 정답 → **1** (지정값)
- 이유: 정답 → **`new int[1]`은 크기만 지정하여 기본값 0으로 채워지고, `new int[]{1}`은 초기화 리스트로 값 1을 직접 지정하여 채워진다**

### 해설
- `new int[1]`: 크기 지정 → Java의 int 기본값 0으로 자동 채움
- `new int[]{1}`: 초기화 리스트 지정 → 명시된 값 1로 채움
- 개발자 생각: 이것이 []과 {}의 본질적 차이를 가장 잘 보여준다. `[]`은 "틀만 만들고 내용은 기본값"이고, `{}`은 "틀에 내용을 직접 주입"한다. Java에서 `{1}`은 단독 사용 불가하며 `new int[]{1}` 형태로 써야 한다 — 선언과 분리된 위치에서는 `new` 키워드가 필수이다.

## 🔑 정답 키워드 요약

| 구분 | `[]` (대괄호) | `{}` (중괄호) |
|------|---------------|---------------|
| **C/Java 의미** | 크기(수량)·인덱스(위치) | 값(내용)·초기화리스트 |
| **Python 의미** | 리스트(list) 타입 | 셋(set) / 딕셔너리(dict) 타입 |
| **직관** | "몇 개? 어디?" | "어떤 값?" |

## AI 프롬프트 작성 가이드

### 프롬프트 작성 핵심 원칙

1. **정답 키워드 명시**: 프롬프트에 `"정답: (키워드)"`을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: `( )`에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.

### 본 주제의 프롬프트 예시

```
정보처리기사 주관식 단답형 문제를 3개 생성하라.

유형: 빈칸 채우기 / 직접 질문 / 비교 설명 (각 1개)
주제: C언어, Java, Python에서 대괄호 []와 중괄호 {}의 의미 차이
정답 키워드: 
  - C/Java: [] = 크기(수량·위치), {} = 값(내용·초기화리스트)
  - Python: [] = 리스트(list), {} = 셋(set) / 딕셔너리(dict)
결정적 구분자: "크기 vs 값" (C/Java), "리스트 vs 셋" (Python)

각 문제에 해설과 개발자 생각(개발적 직관)을 한글로 포함하라.
```
