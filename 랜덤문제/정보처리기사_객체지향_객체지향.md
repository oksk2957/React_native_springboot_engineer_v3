### 정답
**객체지향(Object-Oriented)**

## 객체지향 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **클래스(Class)** | 객체를 생성하기 위한 템플릿/설계도 |
| **객체(Object)** | 클래스의 인스턴스, 실체가 있는 개체 |
| **캡슐화(Encapsulation)** | 데이터와 메서드를 하나로 묶고 외부 접근 제한 |
| **상속(Inheritance)** | 기존 클래스의 속성과 메서드를 재사용/확장 |
| **다형성(Polymorphism)** | 동일한 메서드 호출로 다양한 동작 수행 |
| **추상화(Abstraction)** | 공통 속성과 행위를 추출하여 일반화 |

## 객체지향 vs 절차지향 비교

| 구분 | 객체지향 | 절차지향 |
|------|---------|---------|
| 중심 단위 | 객체(Object) | 함수(Function) |
| 데이터 처리 | 객체가 자신의 데이터 처리 | 함수가 데이터를 처리 |
| 재사용성 | 상속을 통한 높은 재사용성 | 함수 재사용 위주 |
| 유지보수 | 모듈 단위로 독립적 유지보수 | 전체 흐름 파악 필요 |
| 대표 언어 | Java, C++, Python, C# | C, Pascal, Fortran |

## AI 프롬프트 작성 가이드 (랜덤 질문 생성용)

### 프롬프트 작성 핵심 원칙
1. **정답 고정**: 파일명의 마지막 부분(관련정답)이 모든 문제의 고정 정답이 되어야 함
2. **질문 랜덤화**: 같은 정답이라도 질문은 다양한 형태로 랜덤하게 생성
3. **맥락 다양화**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락에서 질문 생성

### 예시 프롬프트
```
정보처리기사 시험을 위한 주관식 단답형 문제를 생성해주세요.

[핵심 규칙]
- 고정 정답: "객체지향(Object-Oriented)"
- 모든 문제의 정답은 반드시 "객체지향(Object-Oriented)"여야 함
- 질문은 다양한 형태로 랜덤하게 생성

[조건]
- 주제: 소프트웨어 공학의 객체지향(Object-Oriented)
- 문제 유형: 주관식 단답형 (빈칸 채우기)
- 난이도: 중급
- 문제 수: 3개
- 고정 정답: 객체지향(Object-Oriented) (모든 문제 동일)

[질문 다양화 예시]
- 문제 1: 정의 관점 (데이터와 메서드를 하나의 단위로 묶는 패러다임)
- 문제 2: 비교 관점 (절차지향과 대비되는 설계 방법론)
- 문제 3: 특징 관점 (캡슐화, 상속, 다형성을 핵심으로 하는 개발 방법)

[출력 형식]
각 문제는 다음 형식으로 작성:
- 질문: 다양한 표현으로 질문
- 정답: 객체지향(Object-Oriented) (고정)
- 해설: 해당 질문의 맥락에 맞는 해설
```

### 프롬프트 작성 시 주의사항
- **정답 고정 확인**: 모든 문제의 정답이 파일명의 관련정답과 일치하는지 검증
- **질문 중복 방지**: 같은 표현의 질문이 반복되지 않도록 다양화
- **맥락 다양성**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락 활용
- **난이도 조절**: 기출문제 수준의 난이도로 설정
- **오답 유도 방지**: 정답이 명확하고 유일하도록 문제 설계
