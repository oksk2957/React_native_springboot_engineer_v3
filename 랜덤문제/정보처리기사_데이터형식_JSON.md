# 정보처리기사 데이터 형식 - JSON 주관식 단답형 문제

> **정답 키워드:** JSON
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** JavaScript 객체 표기법을 기반으로 한 가벼운 데이터 교환 형식으로, 키-값 쌍으로 데이터를 표현하며 웹 API와 RESTful 서비스에서 널리 사용되는 데이터 형식을 **[     ]** 이라고 한다.

**정답:** JSON

---

## 문제 2 (직접 질문 유형)

**질문:** JavaScript의 객체 리터럴 문법을 기반으로 하며, 중괄호({})와 대괄호([])를 사용하여 데이터를 표현하는 가벼운 데이터 교환 형식으로, 웹 브라우저와 서버 간의 데이터 전송에 주로 사용되는 데이터 형식의 이름은 무엇인가?

**정답:** JSON

---

## 문제 3 (빈칸 채우기 + 비교 설명 유형)

**질문:** 데이터 교환 형식 중 **[     ]** 는 XML보다 가벼운 문법으로 데이터를 표현하며, 파싱이 쉽고 대부분의 프로그래밍 언어에서 기본적으로 지원한다. 이 형식은 키-값 쌍으로 데이터를 구성하며, 웹 API 응답과 RESTful 서비스에서 표준으로 사용된다. 반면 XML은 태그 기반의 계층 구조로 더 복잡한 데이터 표현이 가능하다.

**정답:** JSON

---

## AI 프롬프트 작성 가이드

JSON를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: JSON
- 유형: 빈칸 채우기와 직접 질문을 혼합
- JSON의 결정적 키워드(JavaScript 객체 표기법, 키-값 쌍, 중괄호, 대괄호, 웹 API, RESTful 서비스, 데이터 교환, 가벼운 형식)를 문제 설명에 배치하여 정답을 유도
- XML과 비교하여 JSON의 특징을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: JSON"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "키-값 쌍" vs "태그 기반" (XML), "중괄호와 대괄호" vs "태그" (XML), "가벼운 형식" vs "복잡한 계층 구조" (XML), "웹 API 표준" vs "SOAP 웹 서비스" (XML) 등 각 데이터 형식의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
