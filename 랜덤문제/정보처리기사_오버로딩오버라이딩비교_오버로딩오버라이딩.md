# 정보처리기사 오버로딩·오버라이딩 비교 - 오버로딩/오버라이딩 주관식 단답형 문제

> **정답 키워드:** 오버로딩 (Overloading) / 오버라이딩 (Overriding)
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** Java에서 같은 클래스 내에 메서드 이름이 같지만 매개변수의 개수나 타입이 다른 여러 메서드를 정의하는 기법을 **[     ]** 라고 한다. 이 기법은 컴파일 시점에 호출할 메서드가 결정되므로 **정적 바인딩(Static Binding)** 이 적용된다. 예를 들어 `void DoIt()`, `void DoIt(int a)`, `void DoIt(int a, int b)`는 모두 같은 이름이지만 매개변수가 다르므로 별개의 메서드로 인식된다.

**정답:** 오버로딩 (Overloading)

**개발 생각:** 빈칸 채우기 유형은 정보처리기사 실기 시험에서 가장 빈출되는 출제 형식이다. 문제 설명에 "같은 클래스 내", "매개변수의 개수나 타입이 다른", "정적 바인딩", "컴파일 시점"이라는 키워드가 포함되어 있으면, 정답은 오버로딩으로 유도된다. 오버로딩의 핵심 구분자는 **"매개변수 시그니처 차이"**와 **"정적 바인딩(컴파일 시점 결정)"**이다. 오버라이딩은 "부모-자식 관계", "동적 바인딩", "실행 시점 결정"이 키워드이므로, "같은 클래스 내"와 "컴파일 시점"이라는 표현이 오버로딩의 고유 구분자이다. 프롬프트 작성 시 이 키워드를 명시하면 AI가 정답을 오버로딩으로 정확히 유도할 수 있다.

---

## 문제 2 (직접 질문 유형)

**질문:** Java에서 부모 클래스에 정의된 메서드를 자식 클래스가 같은 이름과 같은 매개변수로 다시 정의하여, 부모 메서드의 동작을 덮어쓰고 자식 메서드가 실행되게 하는 기법의 이름은 무엇인가? 이 기법은 실행 시점(Runtime)에 호출할 메서드가 결정되므로 동적 바인딩(Dynamic Binding)이 적용되며, 자식 클래스에서 `@Override` 어노테이션을 사용하여 명시적으로 표시할 수 있다.

**정답:** 오버라이딩 (Overriding)

**개발 생각:** 직접 질문 유형은 객체지향 프로그래밍의 상속 관계에서 메서드 재정의 개념을 직접 묻는 형식이다. "부모 클래스의 메서드를 자식이 다시 정의", "덮어쓰고 자식 메서드가 실행", "동적 바인딩", "실행 시점(Runtime)", "@Override 어노테이션"이라는 구체적 기술 키워드가 오버라이딩을 직접 지칭한다. 오버로딩은 "매개변수 차이", "같은 클래스", "정적 바인딩"이 핵심이므로, "부모-자식 상속 관계"와 "동적 바인딩(실행 시점 결정)"이라는 특성이 오버라이딩 정답 유도에 효과적이다. 프롬프트에서 "부모 메서드를 자식이 재정의, 동적 바인딩, @Override"라고 지정하면 AI가 오버라이딩을 정답으로 생성한다.

---

## 문제 3 (비교 설명 유형)

**질문:** Java의 메서드 재정의 기법 중, **오버로딩(Overloading)**은 같은 클래스 내에서 매개변수 시그니처(개수·타입)만 다르게 여러 메서드를 정의하고 컴파일 시점에 정적 바인딩으로 선택된다. 반면, **[     ]** 은(는) 부모 클래스의 메서드를 자식 클래스가 같은 시그니처로 재정의하여 부모 메서드를 덮어쓰며, 실행 시점에 동적 바인딩으로 선택된다. 두 기법의 결정적 차이는 **"정적 바인딩 vs 동적 바인딩"**과 **"매개변수 차이 vs 시그니처 동일"**이다.

**정답:** 오버라이딩 (Overriding)

**개발 생각:** 비교 설명 유형은 정답 항목(오버라이딩)과 대조 항목(오버로딩)을 동시에 제시하여, 대조 항목의 특성(같은 클래스 내, 매개변수 차이, 정적 바인딩)과 정답 항목의 특성(부모-자식 상속, 시그니처 동일, 동적 바인딩)을 명확히 구분한다. 이 유형은 정보처리기사 시험에서 오버로딩과 오버라이딩의 차이점을 묻는 변형으로 자주 출제된다. 특히 "정적 바인딩 vs 동적 바인딩"과 "매개변수 차이 vs 시그니처 동일"이라는 이중 구분자를 추가하면, 오버로딩과 오버라이딩의 개념 혼동을 방지할 수 있다. 프롬프트 작성 시 "오버로딩과 비교하여 [빈칸]의 특징을 설명하는 문제, 정답은 오버라이딩, 바인딩 시점과 시그니처 구분자 포함"이라고 지정하면, AI가 대조 항목 설명을 배치하고 빈칸에 오버라이딩을 정답으로 유도하는 문제를 정확히 생성한다.

## 한눈에 보는 오버로딩과 오버라이딩 비교표

| 구분 | 오버로딩 (Overloading) | 오버라이딩 (Overriding) |
|------|------------------------|-------------------------|
| 정의 | 같은 이름의 메서드를 매개변수만 다르게 여러 개 정의 | 부모 메서드를 자식이 같은 시그니처로 다시 정의 |
| 클래스 관계 | 같은 클래스 내 | 부모-자식 상속 관계 |
| 매개변수 | 개수 또는 타입이 다름 (시그니처 차이) | 개수·타입·순서 모두 동일 (시그니처 동일) |
| 바인딩 시점 | 컴파일 시점 (정적 바인딩, Static Binding) | 실행 시점 (동적 바인딩, Dynamic Binding) |
| 선택 방식 | 매개변수에 따라 컴파일러가 선택 | 실제 객체 타입에 따라 JVM이 실행 시점에 선택 |
| 어노테이션 | 없음 (명시적 표시 불필요) | `@Override` 사용 가능 |
| 코드 예시 | `void DoIt();` / `void DoIt(int a);` / `void DoIt(int a, int b);` | `class Parent { void DoIt(int a) {} }` → `class Child extends Parent { @Override void DoIt(int a) {} }` |

## AI 프롬프트 작성 가이드

오버로딩·오버라이딩을 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: 오버로딩 (Overloading) / 오버라이딩 (Overriding)
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- 각 문제에 정답과 개발 생각(해설)을 포함
- 오버로딩의 결정적 키워드(같은 클래스 내, 매개변수 시그니처 차이, 정적 바인딩, 컴파일 시점)를 문제 설명에 배치하여 오버로딩 정답을 유도
- 오버라이딩의 결정적 키워드(부모-자식 상속, 시그니처 동일, 동적 바인딩, 실행 시점, @Override)를 문제 설명에 배치하여 오버라이딩 정답을 유도
- 오버로딩과 오버라이딩의 비교 설명을 1개 문제에 포함, "정적 바인딩 vs 동적 바인딩"과 "매개변수 차이 vs 시그니처 동일" 구분자를 명시
- Java 코드 예시를 각 문제에 포함하여 개념 이해를强化
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: 오버로딩 / 오버라이딩"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "정적 바인딩(컴파일 시점) vs 동적 바인딩(실행 시점)"과 "매개변수 차이 vs 시그니처 동일" 등 각 기법의 고유 키워드를 프롬프트에 포함하면, AI가 오버로딩과 오버라이딩을 명확히 구분하는 문제를 생성한다. 특히 바인딩 시점(정적 vs 동적)이 가장 강력한 구분자이다.
4. **개발 생각 요청**: "개발 생각(해설)을 포함"이라고 지정하면, AI가 문제 설계 의도와 출제 패턴 분석을 함께 제공하여 학습 효율이 향상된다.
5. **비교표 포함**: 오버로딩과 오버라이딩은 항상 비교 출제되므로, 비교표를 프롬프트에 포함하면 AI가 두 개념의 전체적 차이를 구조화하여 문제에 반영한다.
