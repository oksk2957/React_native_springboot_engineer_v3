# 정보처리기사 오류검출기법 - CRC

## AI 프롬프트 작성 가이드

- **정답 키워드 명시**: 프롬프트에 "정답: ( )"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
- **유형 지정**: 1.빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제만 랜덤형식의 질문으로 다양성이 보장되어 작성한다. 2.정답은 ()으로 고정된다.
- **결정적 구분자 제시**: ( )에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
- **해설 미포함**: 질문과 정답만 작성하며 해설은 작성하지 않는다.

---

## 문제 1

**유형**: 빈칸 채우기

데이터를 다항식으로 표현하여 나눗셈 연산을 통해 오류를 검출하는 방식으로, Ethernet과 HDLC 등에서 널리 사용되는 오류 검출 기법을 ( )이라고 한다.

**정답**: CRC

---

## 문제 2

**유형**: 직접 질문

데이터 블록을 생성 다항식으로 나누어 나머지를 나타내는 값을 전송하고, 수신 측에서 동일한 다항식으로 나누어 나머지가 0이면 정상으로 판단하는 오류 검출 기법의 영문 약어는 무엇인가?

**정답**: CRC

---

## 문제 3

**유형**: 비교 설명

체크섬은 데이터 값을 단순 합산하여 검사 값을 생성하는 방식이고, ( )는 다항식 기반 나눗셈 연산을 사용하여 오류를 검출하는 방식이다. 후자는 버스트 오류 검출 능력이 우수하여 네트워크 통신에서 널리 사용된다.

**정답**: CRC
