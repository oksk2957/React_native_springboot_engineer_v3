# 정보처리기사 통합테스트 - 상향식통합 주관식 단답형 문제

> **정답 키워드:** 상향식통합
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 통합 테스트 방식 중, 하위 모듈부터 시작하여 상위 모듈로 점진적으로 통합해 나가는 방식을 **[     ]** 이라고 한다. 이 방식은 하위 모듈을 먼저 개발하고 테스트한 후, 상위 모듈을 연결하면서 전체 시스템을 완성해 나간다.

**정답:** 상향식통합

---

## 문제 2 (직접 질문 유형)

**질문:** 통합 테스트 시 하위 모듈을 먼저 개발하고 테스트한 후, 상위 모듈을 연결해 나가는 방식으로, 아직 개발되지 않은 상위 모듈을 대신하는 드라이버(driver)를 사용하는 테스트 방식의 이름은 무엇인가?

**정답:** 상향식통합

---

## 문제 3 (빈칸 채우기 + 비교 설명 유형)

**질문:** 통합 테스트 방식 중, 하위 모듈부터 상위 모듈로 통합해 나가는 **[     ]** 방식은 하위 모듈의 기능을 먼저 검증할 수 있어 초기 단계부터 안정성을 확보할 수 있다. 반면 상위 모듈을 먼저 개발하는 하향식통합과 달리, 최종 시스템의 전체 구조는 후반부에 확인할 수 있다.

**정답:** 상향식통합

---

## AI 프롬프트 작성 가이드

상향식통합을 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: 상향식통합
- 유형: 빈칸 채우기와 직접 질문을 혼합
- 상향식통합의 결정적 키워드(하위 모듈부터 시작, 상위 모듈로 통합, 드라이버 사용, 하향식통합과의 비교)를 문제 설명에 배치하여 정답을 유도
- 하향식통합과 비교하여 상향식통합의 차이점을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: 상향식통합"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "하위 모듈부터 시작" vs "상위 모듈부터 시작" (하향식통합), "드라이버 사용" vs "스텁 사용" (하향식통합) 등 각 방식의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
