# 정보처리기사 SQL - TRUNCATE 주관식 단답형 문제

> **정답 키워드:** TRUNCATE
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 데이터베이스에서 테이블의 모든 데이터를 삭제하되 테이블 구조는 유지하며, DELETE보다 빠르게 처리되고 로그를 최소로 기록하는 SQL 명령어는 **[     ]** 이다.

**정답:** TRUNCATE

---

## 문제 2 (직접 질문 유형)

**질문:** 데이터베이스에서 테이블의 전체 행을 일괄 삭제하되 테이블 구조와 인덱스는 그대로 유지하며, WHERE 조건을 사용할 수 없고 롤백이 불가능한 DDL 성격의 명령어는 무엇인가?

**정답:** TRUNCATE

---

## 문제 3 (비교 설명 유형)

**질문:** 데이터베이스에서 **DELETE**는 WHERE 조건에 따라 특정 행을 삭제하며 트랜잭션 내에서 롤백이 가능하고, 행 단위로 삭제되어 느리다. 반면, **[     ]** 은(는) 테이블의 모든 행을 일괄 삭제하며 테이블 구조는 유지하지만, WHERE 조건을 사용할 수 없고 롤백이 불가능하며 데이터 페이지를 재할당하여 훨씬 빠르게 처리되는 DDL 성격의 명령어이다.

**정답:** TRUNCATE

---

## AI 프롬프트 작성 가이드

TRUNCATE를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: TRUNCATE
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- TRUNCATE의 결정적 키워드(테이블 전체 데이터 삭제, 테이블 구조 유지, DELETE보다 빠름, WHERE 조건 불가, 롤백 불가, DDL 성격, 데이터 페이지 재할당)를 문제 설명에 배치하여 정답을 유도
- DELETE와 TRUNCATE의 차이점을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: TRUNCATE"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "WHERE 조건 불가" vs "WHERE 조건 가능(DELETE)", "롤백 불가" vs "롤백 가능(DELETE)", "DDL 성격" vs "DML 성격(DELETE)", "데이터 페이지 재할당" vs "행 단위 삭제" 등 TRUNCATE의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
