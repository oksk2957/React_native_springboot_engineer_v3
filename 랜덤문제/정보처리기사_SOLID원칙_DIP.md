# 정보처리기사 SOLID 원칙 - DIP(의존관계 역전 원칙) 주관식 단답형 문제

> **출제 범위**: 소프트웨어 공학 - SOLID 원칙 / DIP(Dependency Inversion Principle)
> **문제 유형**: 주관식 단답형 (빈칸 채우기)
> **난이도**: 중급
> **고정 정답**: DIP(의존관계 역전 원칙)

---

## 문제 1

### 질문
상위 모듈은 하위 모듈에 의존해서는 안 되며, 둘 다 추상화에 의존해야 한다는 SOLID 원칙은 무엇인가?

### 정답
**(DIP)**

---

## 문제 2

### 질문
구체적인 구현 클래스가 아닌 인터페이스나 추상 클래스에 의존함으로써 모듈 간의 결합도를 낮추는 객체지향 설계 원칙은 무엇인가?

### 정답
**(DIP)**

---

## 문제 3

### 질문
의존성 주입(Dependency Injection)과 같은 기법을 통해 구현체가 아닌 추상화에 의존하도록 설계하는 SOLID 원칙은 무엇인가?

### 정답
**(DIP)**

---

## DIP(의존관계 역전 원칙) 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **정의** | 상위 모듈과 하위 모듈 모두 추상화에 의존해야 함 |
| **핵심 원칙 1** | 상위 모듈은 하위 모듈에 의존해서는 안 됨 |
| **핵심 원칙 2** | 추상화는 세부 사항에 의존해서는 안 됨 |
| **구현 방법** | 인터페이스/추상클래스 도입, 의존성 주입(DI) 사용 |
| **장점** | 결합도 감소, 테스트 용이성, 유연한 아키텍처 구축 |
| **적용 예시** | 스프링 프레임워크의 DI 컨테이너, Repository 패턴 |

## AI 프롬프트 작성 가이드 (랜덤 질문 생성용)

### 프롬프트 작성 핵심 원칙
1. **정답 고정**: 파일명의 마지막 부분(관련정답)이 모든 문제의 고정 정답이 되어야 함
2. **질문 랜덤화**: 같은 정답이라도 질문은 다양한 형태로 랜덤하게 생성
3. **맥락 다양화**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락에서 질문 생성

- 고정 정답: "DIP(의존관계 역전 원칙)"
- 모든 문제의 정답은 반드시 "DIP(의존관계 역전 원칙)"여야 함
- 질문은 다양한 형태로 랜덤하게 생성

[조건]
- 주제: SOLID 원칙 중 DIP(Dependency Inversion Principle)
- 문제 유형: 주관식 단답형 (빈칸 채우기)
- 난이도: 중급

- 고정 정답: DIP(의존관계 역전 원칙) (모든 문제 동일)

- 문제 1: 정의 관점 (상위 모듈과 하위 모듈이 모두 추상화에 의존해야 하는 원칙)
- 문제 2: 구현 관점 (구체적인 구현이 아닌 인터페이스에 의존하도록 설계하는 원칙)

### 프롬프트 작성 시 주의사항
- **정답 고정 확인**: 모든 문제의 정답이 파일명의 관련정답과 일치하는지 검증
- **질문 중복 방지**: 같은 표현의 질문이 반복되지 않도록 다양화
- **맥락 다양성**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락 활용
- **난이도 조절**: 기출문제 수준의 난이도로 설정
- **오답 유도 방지**: 정답이 명확하고 유일하도록 문제 설계
