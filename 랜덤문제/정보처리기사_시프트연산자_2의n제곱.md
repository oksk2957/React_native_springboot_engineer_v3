# 정보처리기사 시프트연산자 - X << n, X >> n 주관식 단답형 문제

> **정답 키워드:** X × 2ⁿ (왼쪽 시프트), X ÷ 2ⁿ (오른쪽 시프트)
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** C언어의 비트 시프트 연산자 중, `X << n`은 X의 모든 비트를 n칸 **왼쪽**으로 이동시키는 연산이다. 이때 오른쪽 빈 자리는 0으로 채워지며, 수학적으로 `X << n`의 결과값은 **[     ]** 와 동일하다. 예를 들어, `5 << 2`의 결과는 5 × 2² = 5 × 4 = **20**이다.

**정답:** X × 2ⁿ (X를 2의 n제곱으로 곱한 값)

**개발 생각:** 빈칸 채우기 유형은 정보처리기사 실기 시험에서 가장 빈출되는 출제 형식이다. 문제 설명에 "비트를 n칸 왼쪽으로 이동", "오른쪽 빈 자리는 0으로 채워짐"이라는 키워드가 포함되어 있으면, 정답은 X × 2ⁿ으로 유도된다. 시프트 연산의 핵심 원리는 "이동 방향 = 연산 방향"이다. 왼쪽으로 이동하면 값이 커지므로 곱셈(×), 오른쪽으로 이동하면 값이 작아지므로 나눗셈(÷)이다. "왼쪽 = 곱셈"이라는 직관적 연결이 결정적 구분자이다. 예시로 `5 << 2 = 20`을 제시하면, 5 × 4 = 20이라는 계산 과정을 통해 2ⁿ 곱셈 관계를 확인할 수 있다. 프롬프트 작성 시 "왼쪽 시프트의 수학적 의미"를 묻는 빈칸 문제, 정답은 X × 2ⁿ이라고 지정하면 AI가 정확히 유도한다.

---

## 문제 2 (직접 질문 유형)

**질문:** 비트 시프트 연산자 `X >> n`의 수학적 의미를 설명하시오. 이 연산은 X의 모든 비트를 n칸 오른쪽으로 이동시키며, 왼쪽 빈 자리는 양수일 때 0, 음수일 때 1(부호 유지)로 채워진다. 결과값은 소수점 이하를 버림한 정수이다.

**정답:** X ÷ 2ⁿ (X를 2의 n제곱으로 나눈 값, 소수점 이하 버림)

**개발 생각:** 직접 질문 유형은 시프트 연산의 수학적 의미를 직접 묻는 형식으로, 정보처리기사 실기에서 "연산자의 의미를 설명하시오"라는 출제 방식에 대응한다. "비트를 n칸 오른쪽으로 이동", "소수점 이하 버림"이라는 키워드가 X ÷ 2ⁿ을 직접 지칭한다. 오른쪽 시프트의 핵심은 나눗셈이지만, 일반 나눗셈과 달리 "소수점 이하 버림(정수 나눗셈)"이라는 특성이 구분자이다. 예를 들어 `7 >> 1 = 3`인데, 7 ÷ 2 = 3.5에서 소수점을 버린 3이 결과값이다. 음수 시프트 시 부호 유지(산술 시프트) vs 부호 무시(논리 시프트)의 차이도 언어별로 다르지만, 정보처리기사 시험 범위에서는 "부호 유지"가 기본 답안이다. 프롬프트 작성 시 "오른쪽 시프트의 수학적 의미, 소수점 버림 특성 포함"이라고 지정하면 AI가 X ÷ 2ⁿ을 정답으로 생성한다.

---

## 문제 3 (비교 설명 유형)

**질문:** 비트 시프트 연산에서 `X << n`은 비트를 왼쪽으로 n칸 이동하여 **[     ]** 의 결과를 얻고, `X >> n`은 비트를 오른쪽으로 n칸 이동하여 **[     ]** 의 결과를 얻는다. 즉, 왼쪽 시프트는 2의 n제곱 **곱셈** 연산, 오른쪽 시프트는 2의 n제곱 **나눗셈** 연산(소수점 이하 버림)과 각각 대응한다. n=1일 때 X<<1은 X×2, n=2일 때 X<<2는 X×4, n=3일 때 X<<3은 X×8이 되며, 오른쪽 시프트는 반대로 X÷2, X÷4, X÷8이 된다.

**정답:** X × 2ⁿ, X ÷ 2ⁿ

**개발 생각:** 비교 설명 유형은 왼쪽 시프트(<<)와 오른쪽 시프트(>>)를 동시에 제시하여, 두 연산의 대응 관계를 명확히 구분한다. 이 유형은 정보처리기사 시험에서 "두 연산자의 차이점을 비교 설명"하는 변형으로 자주 출제된다. 결정적 구분자는 "이동 방향 ↔ 연산 종류"의 대응: 왼쪽(<<) = 곱셈(×), 오른쪽(>>) = 나눗셈(÷). n값에 따른 계산표(n=1→×2/÷2, n=2→×4/÷4, n=3→×8/÷8)를 문제에 포함하면, 2ⁿ의 지수 증가 패턴을 시각적으로 확인할 수 있어 학습 효율이 높다. 빈칸이 2개이므로 "X × 2ⁿ, X ÷ 2ⁿ"을 순서대로 모두 작성해야 정답 처리된다. 프롬프트 작성 시 "<<와 >>의 수학적 의미를 비교하는 빈칸 2개 문제, 정답은 X × 2ⁿ과 X ÷ 2ⁿ, n값 계산표 포함"이라고 지정하면 AI가 두 연산의 대응 관계를 정확히 생성한다.

---

## AI 프롬프트 작성 가이드

비트 시프트 연산자(X << n, X >> n)를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: X × 2ⁿ (왼쪽 시프트), X ÷ 2ⁿ (오른쪽 시프트)
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- 각 문제에 정답과 개발 생각(해설)을 포함
- 시프트 연산의 결정적 키워드(왼쪽=곱셈, 오른쪽=나눗셈, 2의 n제곱, 소수점 버림, 비트 이동 방향)를 문제 설명에 배치하여 정답을 유도
- <<와 >>의 비교 설명을 1개 문제에 포함, n값 계산표(n=1~3)를 제시
- 오른쪽 시프트의 소수점 이하 버림 특성과 음수 부호 유지 특성을 명시
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: X × 2ⁿ, X ÷ 2ⁿ"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "왼쪽 이동 = 곱셈(×2ⁿ)" vs "오른쪽 이동 = 나눗셈(÷2ⁿ)"이라는 이동 방향-연산 종류 대응 관계를 프롬프트에 포함하면, AI가 <<와 >>를 명확히 구분하는 문제를 생성한다. "소수점 이하 버림"은 >>의 고유 특성으로, 일반 나눗셈과 시프트 나눗셈을 구분하는 구분자이다.
4. **개발 생각 요청**: "개발 생각(해설)을 포함"이라고 지정하면, AI가 문제 설계 의도와 출제 패턴 분석을 함께 제공하여 학습 효율이 향상된다.
