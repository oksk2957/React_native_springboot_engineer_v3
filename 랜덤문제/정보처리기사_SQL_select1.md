# 정보처리기사 SQL - SELECT 1 주관식 단답형 문제

> **정답 키워드:** SELECT 1
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 데이터베이스에서 연결 유효성을 검증하거나 단순 참/거짓 결과를 확인할 목적으로, 실제 테이블 데이터를 조회하지 않고 상수값 1을 반환하는 SQL 구문은 **[     ]** 이다.

**정답:** SELECT 1

---

## 문제 2 (직접 질문 유형)

**질문:** 데이터베이스 연결 풀(Connection Pool)에서 유휴 연결의 유효성을 검사하거나, EXISTS 서브쿼리에서 조건 만족 여부만 확인할 때 사용하는 상수값을 반환하는 SQL 구문의 이름은 무엇인가?

**정답:** SELECT 1

---

## 문제 3 (비교 설명 유형)

**질문:** SQL에서 **SELECT *** 는 실제 테이블의 모든 컬럼 데이터를 반환하며, I/O와 메모리 부하가 크다. 반면, **[     ]** 은(는) 실제 데이터를 조회하지 않고 상수값 1만 반환하므로, 연결 테스트나 EXISTS 서브쿼리에서 성능 부하를 최소화하면서 조건 충족 여부를 확인할 수 있는 구문이다.

**정답:** SELECT 1

---

## AI 프롬프트 작성 가이드

SELECT 1를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: SELECT 1
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- SELECT 1의 결정적 키워드(연결 유효성 검증, 상수값 반환, EXISTS 서브쿼리, 성능 부하 최소화, 연결 풀)를 문제 설명에 배치하여 정답을 유도
- SELECT *와 SELECT 1의 차이점을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: SELECT 1"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "상수값 1 반환" vs "실제 데이터 조회(SELECT *)", "연결 유효성 검증" vs "데이터 조회", "EXISTS 서브쿼리" vs "데이터 반환 쿼리" 등 SELECT 1의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
