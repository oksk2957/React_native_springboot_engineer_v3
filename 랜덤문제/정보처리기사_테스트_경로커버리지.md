# 정보처리기사 테스트 - 경로 커버리지

## AI 프롬프트 작성 가이드

- **정답 키워드 명시**: 프롬프트에 "정답: ( )"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
- **유형 지정**: 1.빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제만 랜덤형식의 질문으로 다양성이 보장되어 작성한다. 2.정답은 ()으로 고정된다.
- **결정적 구분자 제시**: ( )에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
- **해설 미포함**: 질문과 정답만 작성하며 해설은 작성하지 않는다.

---

## 문제 1

**유형**: 빈칸 채우기

프로그램의 모든 가능한 실행 경로(Path)를 최소 한 번 이상 수행하도록 테스트 케이스를 설계하는 가장 엄격한 커버리지 기준을 ( )이라고 한다.

**정답**: 경로 커버리지

---

## 문제 2

**유형**: 직접 질문

경로 커버리지는 프로그램 내의 모든 가능한 경로 조합을 테스트하는 기법으로, 이론적으로 가장 완벽한 테스트를 보장한다. 하지만 루프(Loop)나 조건 분기가 많아질수록 경로의 수가 기하급수적으로 증가하여 현실적으로 모든 경로를 테스트하기 어렵다. 이러한 경로의 총 개수를 계산할 때 사용하는 기본 단위는 무엇인가?

**정답**: 순환 복잡도

---

## 문제 3

**유형**: 비교 설명

구문 커버리지는 모든 코드 라인을 실행하는 데 중점을 두고, 결정 커버리지는 모든 분기의 참/거짓을 커버하며, 조건/결정 커버리지는 개별 조건과 전체 논리식을 모두 검증한다. 그러나 이들 중 어떤 기법도 프로그램의 모든 가능한 실행 경로를 보장하지는 못한다. 예를 들어 `if (a > 0) { if (b > 0) { ... } }`에서 a>0이 참, b>0이 참인 경로와 a>0이 참, b>0이 거짓인 경로, a>0이 거짓인 경로를 모두 테스트해야 하는 가장 엄격한 커버리지 기준은 무엇인가?

**정답**: 경로 커버리지
