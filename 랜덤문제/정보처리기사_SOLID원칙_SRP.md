# 정보처리기사 SOLID 원칙 - SRP(단일 책임 원칙) 주관식 단답형 문제

> **출제 범위**: 소프트웨어 공학 - SOLID 원칙 / SRP(Single Responsibility Principle)
> **문제 유형**: 주관식 단답형 (빈칸 채우기)
> **난이도**: 중급
> **고정 정답**: SRP(단일 책임 원칙)

---

## 문제 1

### 질문
하나의 클래스는 하나의 이유로만 변경되어야 하며, 하나의 클래스가 하나의 책임만 가져야 한다는 SOLID 원칙은 무엇인가?

### 정답
**(SRP)**

---

## 문제 2

### 질문
모듈이나 클래스가 여러 가지 책임을 동시에 수행하지 않고, 단 하나의 책임만 가지도록 설계하는 객체지향 설계 원칙은 무엇인가?

### 정답
**(SRP)**

---

## 문제 3

### 질문
클래스의 변경 이유가 하나뿐이어야 하며, 이를 통해 코드의 응집도를 높이고 복잡도를 낮추는 SOLID 원칙은 무엇인가?

### 정답
**(SRP)**

---

## SRP(단일 책임 원칙) 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **정의** | 하나의 클래스는 하나의 책임만 가져야 함 |
| **핵심 원칙** | 클래스의 변경 이유는 하나뿐이어야 함 |
| **장점** | 응집도 증가, 복잡도 감소, 유지보수 용이성 향상 |
| **위반 시 문제** | 하나의 변경이 여러 부분에 영향을 미침, 버그 발생 위험 증가 |
| **적용 방법** | 클래스 분리, 관심사 분리(Separation of Concerns) |
| **관련 원칙** | 높은 응집도(High Cohesion)와 일치하는 개념 |
