# 정보처리기사 운영체제 - 선점 스케줄링 주관식 단답형 문제

> **정답 키워드:** 선점 스케줄링 (Preemptive Scheduling)
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 운영체제에서 실행 중인 프로세스가 CPU를 강제로 빼앗기고 다른 프로세스에게 CPU를 할당할 수 있는 스케줄링 방식을 **[     ]** 이라고 한다.

**정답:** 선점 스케줄링 (Preemptive Scheduling)

---

## 문제 2 (직접 질문 유형)

**질문:** 운영체제에서 우선순위가 높은 프로세스가 도착하거나 시간 할당량이 만료되면, 현재 실행 중인 프로세스의 CPU 사용을 강제로 중단하고 다른 프로세스에게 CPU를 넘겨주는 스케줄링 방식의 이름은 무엇인가?

**정답:** 선점 스케줄링 (Preemptive Scheduling)

---

## 문제 3 (비교 설명 유형)

**질문:** 운영체제의 CPU 스케줄링 방식 중 **비선점 스케줄링(Non-preemptive Scheduling)**은 프로세스가 CPU를 할당받으면 작업이 완료될 때까지 CPU를 독점하며, 문맥 교환 오버헤드는 적지만 긴급 프로세스 대응이 어렵다. 반면, **[     ]** 은(는) 실행 중인 프로세스의 CPU 사용을 강제로 중단하고 우선순위가 높은 프로세스나 시간 할당량이 만료된 경우 다른 프로세스에게 CPU를 할당하는 방식으로, 응답 시간은 짧지만 문맥 교환 오버헤드가 크다.

**정답:** 선점 스케줄링 (Preemptive Scheduling)

---

## AI 프롬프트 작성 가이드

선점 스케줄링를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: 선점 스케줄링 (Preemptive Scheduling)
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- 선점 스케줄링의 결정적 키워드(CPU 강제 빼앗기, 우선순위 높은 프로세스, 시간 할당량 만료, 문맥 교환 오버헤드, 응답 시간 짧음, 긴급 프로세스 대응)를 문제 설명에 배치하여 정답을 유도
- 비선점 스케줄링과 선점 스케줄링의 차이점을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: 선점 스케줄링 (Preemptive Scheduling)"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "CPU 강제 빼앗기" vs "CPU 독점(비선점)", "우선순위 높은 프로세스 대응" vs "순차 실행", "응답 시간 짧음" vs "응답 시간 길음", "문맥 교환 오버헤드 큼" vs "문맥 교환 오버헤드 작음" 등 선점 스케줄링의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
