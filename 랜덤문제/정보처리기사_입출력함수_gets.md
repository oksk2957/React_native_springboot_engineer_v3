# 정보처리기사 C언어 입출력함수 - gets() 주관식 단답형 문제

> **정답 키워드:** gets()
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** C언어에서 표준 입력(stdin)으로부터 문자열을 읽어들일 때, 공백(whitespace)을 포함한 전체 줄을 하나의 문자열로 저장하는 함수는 **[     ]** 이다. 이 함수는 줄바꿈 문자(newline)를 읽으면 저장을 종료하고, 줄바꿈 문자는 널 문자('\0')로 변환하여 저장한다.

**정답:** gets()

---

## 문제 2 (직접 질문 유형)

**질문:** C언어의 문자열 입력 함수 중, 버퍼 오버플로우(buffer overflow) 취약점이 있어 C11 표준에서 완전히 제거된 함수의 이름은 무엇인가? 이 함수는 입력 길이를 검사하지 않아 지정된 버퍼 크기를 초과하는 데이터가 들어올 경우 메모리 침범이 발생할 수 있다.

**정답:** gets()

---

## 문제 3 (빈칸 채우기 + 비교 설명 유형)

**질문:** C언어 문자열 입력 함수 중 scanf()는 공백을 구분자로 처리하여 공백 이전까지만 읽지만, **[     ]** 함수는 공백과 탭을 포함한 전체 줄을 하나의 문자열로 읽어들인다. 그러나 후자는 버퍼 크기를 검사하지 않아 오버플로우 위험이 있으며, C11에서는 안전한 대안 함수인 fgets()로 대체되었다.

**정답:** gets()

---

## AI 프롬프트 작성 가이드

gets()를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: gets()
- 유형: 빈칸 채우기와 직접 질문을 혼합
- gets()의 결정적 키워드(공백 포함 전체 줄 입력, 버퍼 오버플로우, C11 제거, fgets() 대체)를 문제 설명에 배치하여 정답을 유도
- scanf()와 비교하여 gets()의 차이점을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: gets()"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "공백 포함 전체 줄" vs "공백 구분자" (scanf), "버퍼 오버플로우" vs "길이 검사" (fgets) 등 각 함수의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
