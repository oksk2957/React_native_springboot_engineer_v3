## 스트래터지(Strategy) 패턴 핵심 개념 정리

| 구성 요소 | 설명 |
|-----------|------|
| **전략(Strategy) 인터페이스** | 모든 구체 전략이 구현하는 공통 인터페이스 |
| **구체 전략(ConcreteStrategy)** | 실제 알고리즘을 구현하는 구체 클래스 |
| **컨텍스트(Context)** | 전략 객체를 사용하는 클라이언트, 실행 환경 제공 |

### 주요 특징
- **알고리즘 캡슐화**: 각 알고리즘을 독립적인 클래스로 분리
- **상호 교환 가능**: 실행 시점에 알고리즘을 동적으로 변경 가능
- **조건문 제거**: if-else, switch 문을 객체 지향적으로 대체
- **OCP 준수**: 새로운 전략 추가 시 기존 코드 수정 없이 확장 가능
- **대표 예시**: Java의 Comparator 인터페이스, 결제 수단 선택, 정렬 알고리즘 교체

## AI 프롬프트 작성 가이드 (랜덤 질문 생성용)

### 프롬프트 작성 핵심 원칙
1. **정답 고정**: 파일명의 마지막 부분(관련정답)이 모든 문제의 고정 정답이 되어야 함
2. **질문 랜덤화**: 같은 정답이라도 질문은 다양한 형태로 랜덤하게 생성
3. **맥락 다양화**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락에서 질문 생성

### 예시 프롬프트
```
정보처리기사 시험을 위한 주관식 단답형 문제를 생성해주세요.

[핵심 규칙]
- 고정 정답: "스트래터지 패턴(Strategy Pattern)"
- 모든 문제의 정답은 반드시 "스트래터지 패턴(Strategy Pattern)"이어야 함
- 질문은 다양한 형태로 랜덤하게 생성

[조건]
- 주제: 디자인 패턴 중 스트래터지 패턴(Strategy Pattern)
- 문제 유형: 주관식 단답형 (빈칸 채우기)
- 난이도: 중급
- 문제 수: 3개
- 고정 정답: 스트래터지 패턴(Strategy Pattern) (모든 문제 동일)

[질문 다양화 예시]
- 문제 1: 정의 관점 (알고리즘군을 정의하고 캡슐화하여 상호 교환 가능하게 만드는 패턴)
- 문제 2: 구조 관점 (동일한 인터페이스를 통해 서로 다른 알고리즘을 실행하는 패턴)
- 문제 3: 장점 관점 (조건문을 제거하고 유연한 확장을 가능하게 하는 패턴)

[출력 형식]
각 문제는 다음 형식으로 작성:
- 질문: 다양한 표현으로 질문
- 정답: 스트래터지 패턴(Strategy Pattern) (고정)
```

### 프롬프트 작성 시 주의사항
- **정답 고정 확인**: 모든 문제의 정답이 파일명의 관련정답과 일치하는지 검증
- **질문 중복 방지**: 같은 표현의 질문이 반복되지 않도록 다양화
- **맥락 다양성**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락 활용
- **난이도 조절**: 기출문제 수준의 난이도로 설정
- **오답 유도 방지**: 정답이 명확하고 유일하도록 문제 설계
