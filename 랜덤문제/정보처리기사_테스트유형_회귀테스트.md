# 정보처리기사 테스트유형 - 회귀테스트 주관식 단답형 문제

> **정답 키워드:** 회귀테스트
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 소프트웨어 테스트에서 기존에 정상적으로 작동하던 기능이 수정이나 변경 후에도 여전히 정상적으로 작동하는지 확인하기 위해 수행하는 테스트를 **[     ]** 라고 한다. 이 테스트는 새로운 결함이 기존 기능에 영향을 주지 않았는지 검증하는 것을 목적으로 한다.

**정답:** 회귀테스트

---

## 문제 2 (직접 질문 유형)

**질문:** 소프트웨어 유지보수 단계에서 코드 변경, 버그 수정, 기능 추가 등으로 인해 기존에 정상적으로 작동하던 기능이 손상되지 않았는지 확인하기 위해 수행하는 테스트의 이름은 무엇인가?

**정답:** 회귀테스트

---

## 문제 3 (빈칸 채우기 + 비교 설명 유형)

**질문:** 소프트웨어 테스트에서 단위 테스트, 통합 테스트, 시스템 테스트 등은 새로운 기능이나 모듈의 정확성을 검증하는 반면, **[     ]** 는 기존에 정상적으로 작동하던 기능이 변경 후에도 여전히 정상적으로 작동하는지 확인하는 테스트이다. 이 테스트는 자동화 도구를 활용하여 반복적으로 수행하는 것이 일반적이다.

**정답:** 회귀테스트

---

## AI 프롬프트 작성 가이드

회귀테스트를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: 회귀테스트
- 유형: 빈칸 채우기와 직접 질문을 혼합
- 회귀테스트의 결정적 키워드(기존 기능 정상 작동 확인, 수정 후 검증, 자동화, 유지보수 단계)를 문제 설명에 배치하여 정답을 유도
- 단위 테스트, 통합 테스트와 비교하여 회귀테스트의 차이점을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: 회귀테스트"를 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "기존 기능 정상 작동 확인" vs "새로운 기능 검증" (단위/통합 테스트), "수정 후 검증" vs "초기 개발 시 검증" 등 각 테스트 유형의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
