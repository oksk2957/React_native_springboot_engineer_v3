# 정보처리기사 ASCII 코드 - 문자와 숫자 매핑 주관식 단답형 문제

> **정답 키워드:** '0'은 48, 'a'는 97
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** ASCII 코드에서 영문 대문자 'A'는 65, 'B'는 66으로 순차적으로 부여된다. 소문자 'a'는 **[     ]** 로 시작하며, 숫자 문자 '0'은 **[     ]** 로 시작한다. 즉, 소문자는 대문자보다 32만큼 큰 값을 가지며, 숫자 문자는 영문자보다 작은 값을 가진다.

**정답:** 97, 48

---

## 문제 2 (직접 질문 유형)

**질문:** ASCII 코드에서 문자 '0'(숫자 문자)에 매핑된 정수값과 문자 'a'(소문자)에 매핑된 정수값을 각각 쓰시오. ASCII 코드는 각 문자에 고유한 정수값을 부여하는 국제 표준이며, 대문자 'A'는 65부터 시작하고 소문자는 대문자보다 32만큼 큰 값에서 시작하며, 숫자 문자는 영문자보다 작은 값에서 시작한다.

**정답:** '0' = 48, 'a' = 97

---

## 문제 3 (비교 설명 유형)

**질문:** ASCII 코드에서 영문 대문자('A'~'Z'), 소문자('a'~'z'), 숫자 문자('0'~'9')의 시작 값을 비교 설명하시오. 대문자 'A'는 65부터 시작하며 소문자 'a'는 **[     ]** 부터 시작하여 대문자보다 32만큼 크다. 숫자 문자 '0'은 **[     ]** 부터 시작하여 영문자보다 작은 값을 가진다. 이는 ASCII 코드가 문자를 순차적으로 배열할 때 대소문자 구분과 숫자 우선순위를 고려한 결과이다.

**정답:** 97, 48

---

## AI 프롬프트 작성 가이드

ASCII 코드의 "'0'은 48, 'a'는 97"을 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: '0'은 48, 'a'는 97
- 정답 설명: ASCII 코드에서 숫자 문자 '0'은 48부터, 소문자 'a'는 97부터 시작
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- ASCII 코드의 결정적 키워드('A'=65, 'a'=97, '0'=48, 대소문자 차이 32, 숫자 문자 우선)를 문제 설명에 배치하여 정답을 유도
- 대문자/소문자/숫자 문자의 시작 값 비교 설명을 1개 문제에 포함
- "'0'을 0으로 오해"라는 오답과 "'0'은 48이다"라는 정답을 구분하는 결정적 구분자를 명시
- "소문자 = 대문자 + 32"라는 규칙을 활용한 문제 1개 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: '0'은 48, 'a'는 97"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "A = 65(대문자 시작)" vs "a = 97(소문자 시작, A+32)" vs "0 = 48(숫자 시작, 영문자보다 작음)"이라는 값 대비를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다. "'0'을 0으로 오해"라는 흔한 오답과 "'0'은 48이다"라는 정답의 구분이 핵심 구분자이다.
