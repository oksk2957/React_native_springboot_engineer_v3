### 정답
**(OCP)**

### 정답
**(OCP)**

## 문제 3

### 정답
**(OCP)**

---

## OCP(개방-폐쇄 원칙) 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **개방(Open)** | 확장에 대해 열려 있음 - 새로운 기능 추가 가능 |
| **폐쇄(Closed)** | 변경에 대해 닫혀 있음 - 기존 코드 수정 불필요 |
| **구현 방법** | 추상화(인터페이스/추상클래스) + 다형성 + 상속 |
| **장점** | 유지보수성 향상, 재사용성 증가, 버그 발생 위험 감소 |
| **적용 예시** | 전략 패턴(Strategy Pattern), 템플릿 메서드 패턴 |

## AI 프롬프트 작성 가이드 (랜덤 질문 생성용)

### 프롬프트 작성 핵심 원칙
1. **정답 고정**: 파일명의 마지막 부분(관련정답)이 모든 문제의 고정 정답이 되어야 함
2. **질문 랜덤화**: 같은 정답이라도 질문은 다양한 형태로 랜덤하게 생성
3. **맥락 다양화**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락에서 질문 생성

### 예시 프롬프트
```
정보처리기사 시험을 위한 주관식 단답형 문제를 생성해주세요.

[핵심 규칙]
- 고정 정답: "OCP(개방-폐쇄 원칙)"
- 모든 문제의 정답은 반드시 "OCP(개방-폐쇄 원칙)"여야 함
- 질문은 다양한 형태로 랜덤하게 생성

[조건]
- 주제: SOLID 원칙 중 OCP(Open-Closed Principle)
- 문제 유형: 주관식 단답형 (빈칸 채우기)
- 난이도: 중급
- 문제 수: 3개
- 고정 정답: OCP(개방-폐쇄 원칙) (모든 문제 동일)

[질문 다양화 예시]
- 문제 1: 정의 관점 (확장에는 열려 있고 변경에는 닫혀 있는 원칙)
- 문제 2: 특징 관점 (기존 코드 수정 없이 기능 추가 가능한 설계 원칙)
- 문제 3: 구현 관점 (추상화와 다형성을 활용하여 기능을 확장하는 원칙)

[출력 형식]
각 문제는 다음 형식으로 작성:
- 질문: 다양한 표현으로 질문
- 정답: OCP(개방-폐쇄 원칙) (고정)
```

### 프롬프트 작성 시 주의사항
- **정답 고정 확인**: 모든 문제의 정답이 파일명의 관련정답과 일치하는지 검증
- **질문 중복 방지**: 같은 표현의 질문이 반복되지 않도록 다양화
- **맥락 다양성**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락 활용
- **난이도 조절**: 기출문제 수준의 난이도로 설정
- **오답 유도 방지**: 정답이 명확하고 유일하도록 문제 설계
