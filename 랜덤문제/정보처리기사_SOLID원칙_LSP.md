# 정보처리기사 SOLID 원칙 - LSP(리스코프 치환 원칙) 주관식 단답형 문제

> **출제 범위**: 소프트웨어 공학 - SOLID 원칙 / LSP(Liskov Substitution Principle)

### 정답
**(LSP)**

## 문제 2

### 질문
자식 클래스는 부모 클래스의 기능을 완전하게 대체할 수 있어야 하며, 부모 클래스의 계약(Contract)을 위반하지 않아야 하는 객체지향 설계 원칙은 무엇인가?

### 정답
**(LSP)**

---

## 문제 3

### 질문
상속 관계에서 하위 클래스가 상위 클래스의 모든 행위를 대체할 수 있어야 하며, 이를 위반할 경우 상속의 잘못된 사용을 의미하는 SOLID 원칙은 무엇인가?

### 정답
**(LSP)**

---

## LSP(리스코프 치환 원칙) 핵심 개념 정리

| 개념 | 설명 |
|------|------|
| **정의** | 상위 클래스의 객체를 하위 클래스의 객체로 치환해도 프로그램의 정확성이 유지되어야 함 |
| **핵심 조건** | 하위 클래스는 상위 클래스의 계약(Contract)을 준수해야 함 |
| **위반 시 문제** | 다형성 사용 시 예상치 못한 버그 발생, 코드 신뢰성 저하 |
| **적용 방법** | 상속 전 "is-a" 관계 확인, 계약에 따른 설계(Design by Contract) |
| **관련 원칙** | OCP(개방-폐쇄 원칙)와 밀접한 관계 |

## AI 프롬프트 작성 가이드 (랜덤 질문 생성용)

### 프롬프트 작성 핵심 원칙
1. **정답 고정**: 파일명의 마지막 부분(관련정답)이 모든 문제의 고정 정답이 되어야 함
2. **질문 랜덤화**: 같은 정답이라도 질문은 다양한 형태로 랜덤하게 생성
3. **맥락 다양화**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락에서 질문 생성

### 프롬프트 작성 시 주의사항
- **정답 고정 확인**: 모든 문제의 정답이 파일명의 관련정답과 일치하는지 검증
- **질문 중복 방지**: 같은 표현의 질문이 반복되지 않도록 다양화
- **맥락 다양성**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락 활용
- **난이도 조절**: 기출문제 수준의 난이도로 설정
- **오답 유도 방지**: 정답이 명확하고 유일하도록 문제 설계
