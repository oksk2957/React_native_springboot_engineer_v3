# 정보처리기사 소프트웨어분류 - 그룹웨어(Groupware) 주관식 단답형 문제

> **정답 키워드:** 그룹웨어 (Groupware)
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 소프트웨어 유형 중, 조직 내 여러 사용자가 협업하여 업무를 수행할 수 있도록 전자우편, 일정관리, 문서관리, 화상회의 등의 기능을 통합 제공하는 소프트웨어를 **[     ]** 라고 한다. 이 소프트웨어는 그룹(group)의 협업을 지원하는 도구(ware)로, 조직의 생산성 향상과 업무 프로세스 효율화를 목표로 한다.

**정답:** 그룹웨어 (Groupware)

**개발 생각:** 빈칸 채우기 유형은 정보처리기사 실기 시험에서 가장 빈출되는 출제 형식이다. 문제 설명에 "조직 내 여러 사용자 협업", "전자우편·일정관리·문서관리·화상회의", "그룹 협업 지원", "생산성 향상"이라는 키워드가 포함되어 있으면, 정답은 그룹웨어로 유도된다. 그룹웨어의 어원인 'group(그룹)' + 'ware(소프트웨어)'의 직관적 메타포가 결정적 구분자이다. ERP는 "전사적 자원관리", CRM은 "고객관계관리", SCM은 "공급망관리"가 키워드이므로, "조직 내 협업 기능 통합 제공"이라는 표현이 그룹웨어의 고유 구분자이다. 프롬프트 작성 시 이 키워드를 명시하면 AI가 정답을 그룹웨어로 정확히 유도할 수 있다.

---

## 문제 2 (직접 질문 유형)

**질문:** 조직 내 여러 구성원이 전자우편, 일정관리, 문서공유, 화상회의 등을 통해 실시간으로 협업할 수 있도록 지원하는 통합 업무 지원 소프트웨어의 이름은 무엇인가?

**정답:** 그룹웨어 (Groupware)

**개발 생각:** 직접 질문 유형은 특정 협업 시나리오(조직 내 구성원 간 실시간 협업)와 기능(전자우편·일정관리·문서공유·화상회의)을 연결하여 정답을 유도한다. "조직 내 여러 구성원", "전자우편·일정관리·문서공유·화상회의", "실시간 협업 지원"이라는 구체적 기술 키워드가 그룹웨어를 직접 지칭한다. ERP는 전사적 자원관리가 핵심이고, CRM은 고객관계관리가 핵심이므로, "조직 내 협업 기능 통합"이라는 업무 지원 특성이 그룹웨어 정답 유도에 효과적이다. 프롬프트에서 "조직 내 협업 기능 통합 업무 지원 소프트웨어"라고 지정하면 AI가 그룹웨어를 정답으로 생성한다.

---

## 문제 3 (비교 설명 유형)

**질문:** 기업용 소프트웨어 유형 중, **ERP(Enterprise Resource Planning)**는 기업의 전반적인 자원을 통합 관리하는 시스템이고, **CRM(Customer Relationship Management)**는 고객과의 관계를 관리하는 시스템이다. 반면, **[     ]** 은(는) 조직 내 여러 사용자가 전자우편, 일정관리, 문서관리, 화상회의 등을 통해 협업할 수 있도록 지원하는 소프트웨어로, 그룹(group)의 협업을 지원하는 도구(ware)의 의미를 담고 있다.

**정답:** 그룹웨어 (Groupware)

**개발 생각:** 비교 설명 유형은 정답 항목과 대조 항목(ERP, CRM)을 동시에 제시하여, 대조 항목의 특성(전사적 자원관리, 고객관계관리)과 정답 항목의 특성(조직 내 협업 지원, 전자우편·일정관리·문서관리·화상회의 통합)을 명확히 구분한다. 이 유형은 정보처리기사 시험에서 기업용 소프트웨어 분류 용어들의 차이점을 묻는 변형으로 자주 출제된다. 특히 "어원에 'group(그룹)' + 'ware(도구)'가 포함"이라는 메타포적 구분자를 추가하면, 그룹웨어와 ERP·CRM의 기능적 차이를 명확히 할 수 있다. 프롬프트 작성 시 "ERP·CRM과 비교하여 [빈칸]의 특징을 설명하는 문제, 정답은 그룹웨어, 어원 구분자 포함"이라고 지정하면, AI가 대조 항목 설명을 배치하고 빈칸에 그룹웨어를 정답으로 유도하는 문제를 정확히 생성한다.

## AI 프롬프트 작성 가이드

그룹웨어를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: 그룹웨어 (Groupware)
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- 각 문제에 정답과 개발 생각(해설)을 포함
- 그룹웨어의 결정적 키워드(조직 내 협업, 전자우편·일정관리·문서관리·화상회의, 생산성 향상, group+ware 어원)를 문제 설명에 배치하여 정답을 유도
- ERP, CRM 등 다른 기업용 소프트웨어 분류 용어와의 비교 설명을 1개 문제에 포함
- 그룹웨어와 ERP·CRM의 기능적 유사성으로 인한 혼동 방지를 위해 어원 구분자(groupware vs enterprise/customer)를 명시
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: 그룹웨어"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "조직 내 협업 기능 통합" vs "전사적 자원관리(ERP)" vs "고객관계관리(CRM)" 등 각 분류 용어의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다. 특히 그룹웨어와 ERP·CRM의 혼동 방지를 위해 어원(groupware vs enterprise/customer)을 구분자로 활용한다.
4. **개발 생각 요청**: "개발 생각(해설)을 포함"이라고 지정하면, AI가 문제 설계 의도와 출제 패턴 분석을 함께 제공하여 학습 효율이 향상된다.
