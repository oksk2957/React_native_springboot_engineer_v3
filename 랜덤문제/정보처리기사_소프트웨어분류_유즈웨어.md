# 정보처리기사 소프트웨어분류 - 유즈웨어(Useware) 주관식 단답형 문제

> **정답 키워드:** 유즈웨어 (Useware)
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 소프트웨어 개발 방법론 중, 사용자의 요구사항과 사용성(usability)을 최우선으로 고려하여 설계·개발되는 소프트웨어를 **[     ]** 라고 한다. 이 소프트웨어는 기술 중심의 개발에서 벗어나 사용자의 업무 환경, 인지적 특성, 사용 편의성을 중심으로 개발되며, 사용자 인터페이스(UI)와 사용자 경험(UX)의 최적화를 핵심 목표로 한다.

**정답:** 유즈웨어 (Useware)

**개발 생각:** 빈칸 채우기 유형은 정보처리기사 실기 시험에서 가장 빈출되는 출제 형식이다. 문제 설명에 "사용자 요구사항 최우선", "사용성(usability)", "사용자 인터페이스(UI)", "사용자 경험(UX)"이라는 키워드가 포함되어 있으면, 정답은 유즈웨어로 유도된다. 유즈웨어의 어원인 'use(사용)' + 'ware(소프트웨어)'의 직관적 메타포가 결정적 구분자이다. 하드웨어는 "물리적 장비", 소프트웨어는 "프로그램·데이터", 펌웨어는 "하드웨어에 내장된 소프트웨어"가 키워드이므로, "사용자 중심 설계·개발"이라는 표현이 유즈웨어의 고유 구분자이다. 프롬프트 작성 시 이 키워드를 명시하면 AI가 정답을 유즈웨어로 정확히 유도할 수 있다.

---

## 문제 2 (직접 질문 유형)

**질문:** 기술 중심의 개발에서 벗어나 사용자의 업무 환경, 인지적 특성, 사용 편의성을 최우선으로 고려하여 사용자 인터페이스(UI)와 사용자 경험(UX)의 최적화를 목표로 개발되는 소프트웨어의 이름은 무엇인가?

**정답:** 유즈웨어 (Useware)

**개발 생각:** 직접 질문 유형은 특정 개발 패러다임(사용자 중심 개발)과 목표(UI·UX 최적화)를 연결하여 정답을 유도한다. "사용자 업무 환경 고려", "인지적 특성 반영", "사용 편의성 최우선", "UI·UX 최적화"라는 구체적 기술 키워드가 유즈웨어를 직접 지칭한다. 하드웨어는 물리적 장비가 핵심이고, 소프트웨어는 프로그램 자체가 핵심이므로, "사용자 중심 설계·개발"이라는 개발 철학의 특성이 유즈웨어 정답 유도에 효과적이다. 프롬프트에서 "사용자 중심 UI·UX 최적화 소프트웨어"라고 지정하면 AI가 유즈웨어를 정답으로 생성한다.

---

## 문제 3 (비교 설명 유형)

**질문:** 소프트웨어 개발 관련 용어 중, **하드웨어(Hardware)**는 컴퓨터의 물리적 장치를 의미하고, **소프트웨어(Software)**는 컴퓨터에서 실행되는 프로그램과 데이터를 의미한다. 반면, **[     ]** 은(는) 기술 중심의 개발에서 벗어나 사용자의 요구사항과 사용성(usability)을 최우선으로 고려하여 사용자 인터페이스(UI)와 사용자 경험(UX)의 최적화를 목표로 개발되는 소프트웨어이다. 이 용어의 어원은 'use(사용)'에서 유래하였다.

**정답:** 유즈웨어 (Useware)

**개발 생각:** 비교 설명 유형은 정답 항목과 대조 항목(하드웨어, 소프트웨어)을 동시에 제시하여, 대조 항목의 특성(물리적 장비, 프로그램·데이터)과 정답 항목의 특성(사용자 중심 설계·개발, UI·UX 최적화)을 명확히 구분한다. 이 유형은 정보처리기사 시험에서 소프트웨어 관련 용어들의 차이점을 묻는 변형으로 자주 출제된다. 특히 "어원에 'use(사용)'가 포함"이라는 메타포적 구분자를 추가하면, 유즈웨어와 소프트웨어(Software)의 기능적 차이를 명확히 할 수 있다. 프롬프트 작성 시 "하드웨어·소프트웨어와 비교하여 [빈칸]의 특징을 설명하는 문제, 정답은 유즈웨어, 어원 구분자 포함"이라고 지정하면, AI가 대조 항목 설명을 배치하고 빈칸에 유즈웨어를 정답으로 유도하는 문제를 정확히 생성한다.

---

## AI 프롬프트 작성 가이드

유즈웨어를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: 유즈웨어 (Useware)
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- 각 문제에 정답과 개발 생각(해설)을 포함
- 유즈웨어의 결정적 키워드(사용자 요구사항 최우선, 사용성 usability, 사용자 인터페이스 UI, 사용자 경험 UX, use 어원)를 문제 설명에 배치하여 정답을 유도
- 하드웨어, 소프트웨어 등 다른 소프트웨어 관련 용어와의 비교 설명을 1개 문제에 포함
- 유즈웨어와 소프트웨어의 기능적 유사성으로 인한 혼동 방지를 위해 어원 구분자(use vs soft)를 명시
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: 유즈웨어"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "사용자 요구사항 최우선" vs "물리적 장비(하드웨어)" vs "프로그램·데이터(소프트웨어)" 등 각 분류 용어의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다. 특히 유즈웨어와 소프트웨어의 혼동 방지를 위해 어원(use vs soft)을 구분자로 활용한다.
4. **개발 생각 요청**: "개발 생각(해설)을 포함"이라고 지정하면, AI가 문제 설계 의도와 출제 패턴 분석을 함께 제공하여 학습 효율이 향상된다.
