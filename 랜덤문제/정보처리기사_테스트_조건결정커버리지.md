# 정보처리기사 테스트 - 조건/결정 커버리지

## AI 프롬프트 작성 가이드

- **정답 키워드 명시**: 프롬프트에 "정답: ( )"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
- **유형 지정**: 1.빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제만 랜덤형식의 질문으로 다양성이 보장되어 작성한다. 2.정답은 ()으로 고정된다.
- **결정적 구분자 제시**: ( )에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
- **해설 미포함**: 질문과 정답만 작성하며 해설은 작성하지 않는다.

---

## 문제 1

**유형**: 빈칸 채우기

개별 조건의 참(True)/거짓(False)과 전체 논리식의 결과(참/거짓)를 모두 한 번 이상 수행하도록 테스트 케이스를 설계하는 커버리지 기준을 ( )이라고 한다.

**정답**: 조건/결정 커버리지

---

## 문제 2

**유형**: 직접 질문

조건/결정 커버리지는 조건 커버리지와 결정 커버리지의 장점을 모두 결합한 기법이다. 이 커버리지 기준을 만족하려면 개별 조건의 모든 가능한 조합을 테스트할 필요는 없지만, 각 조건이 독립적으로 참과 거짓을 가지면서 전체 결정 결과도 참과 거짓을 모두 커버해야 한다. 이러한 커버리지를 영문으로 표현하면 무엇인가?

**정답**: Condition/Decision Coverage

---

## 문제 3

**유형**: 비교 설명

구문 커버리지는 모든 코드 라인을 실행하는 데 중점을 두고, 조건 커버리지는 개별 조건의 참/거짓을 모두 테스트하지만 전체 논리식 결과는 보장하지 못한다. 반면에 조건/결정 커버리지는 이 두 가지를 모두 만족시키는데, 예를 들어 `if (a > 0 || b > 0)`에서 a>0이 참이면 b>0은 평가되지 않는 단축 평가(Short-circuit) 상황에서도 개별 조건과 전체 결과를 모두 커버한다. 조건/결정 커버리지가 결정 커버리지에 비해 갖는 추가적인 테스트 측면은 무엇인가?

**정답**: 개별 조건의 독립적 검증
