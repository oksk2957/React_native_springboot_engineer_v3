# 정보처리기사 데이터 형식 - XML 주관식 단답형 문제

> **정답 키워드:** XML
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 마크업 언어의 일종으로, 태그를 사용하여 데이터의 구조와 의미를 정의하는 데이터 형식을 **[     ]** 이라고 한다. 이 형식은 계층적인 트리 구조로 데이터를 표현하며, 다양한 플랫폼 간의 데이터 교환에 널리 사용된다.

**정답:** XML

---

## 문제 2 (직접 질문 유형)

**질문:** HTML과 유사하게 태그를 사용하여 데이터를 구조화하는 마크업 언어로, 데이터의 의미와 구조를 동시에 표현할 수 있으며, 웹 서비스(SOAP), 설정 파일, 문서 교환 등에 사용되는 데이터 형식의 이름은 무엇인가?

**정답:** XML

---

## 문제 3 (빈칸 채우기 + 비교 설명 유형)

**질문:** 데이터 교환 형식 중 **[     ]** 는 태그 기반의 계층 구조로 데이터를 표현하며, DTD나 XML Schema를 통해 데이터의 유효성을 검증할 수 있다. 반면 JSON은 더 가벼운 형식으로 데이터를 표현하며, 웹 API에서 주로 사용된다.

**정답:** XML

---

## AI 프롬프트 작성 가이드

XML를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: XML
- 유형: 빈칸 채우기와 직접 질문을 혼합
- XML의 결정적 키워드(마크업 언어, 태그, 계층 구조, 데이터 교환, DTD, XML Schema, SOAP, 웹 서비스)를 문제 설명에 배치하여 정답을 유도
- JSON과 비교하여 XML의 특징을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: XML"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "태그 기반 계층 구조" vs "키-값 쌍" (JSON), "DTD/XML Schema로 유효성 검증" vs "스키마 없음" (JSON), "SOAP 웹 서비스" vs "REST API" (JSON) 등 각 데이터 형식의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
