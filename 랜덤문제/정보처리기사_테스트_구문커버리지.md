# 정보처리기사 테스트 - 구문 커버리지

## AI 프롬프트 작성 가이드

- **정답 키워드 명시**: 프롬프트에 "정답: ( )"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
- **유형 지정**: 1.빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제만 랜덤형식의 질문으로 다양성이 보장되어 작성한다. 2.정답은 ()으로 고정된다.
- **결정적 구분자 제시**: ( )에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
- **해설 미포함**: 질문과 정답만 작성하며 해설은 작성하지 않는다.

---

## 문제 1

**유형**: 빈칸 채우기

프로그램의 모든 실행 가능한 구문(Statement)을 최소 한 번 이상 수행하도록 테스트 케이스를 설계하는 커버리지 기준을 ( )이라고 한다.

**정답**: 구문 커버리지

---

## 문제 2

**유형**: 직접 질문

구문 커버리지는 소스 코드의 모든 실행 구문을 한 번 이상 실행하지만, 조건 분기의 참(True)과 거짓(False)을 모두 검증하지 못하는 한계가 있다. 이러한 구문 커버리지의 또 다른 이름은 무엇인가?

**정답**: 라인 커버리지

---

## 문제 3

**유형**: 비교 설명

구문 커버리지는 코드의 모든 라인을 실행하는 데는 유효하지만, 조건식의 논리적 오류를 발견하지 못할 수 있다. 예를 들어 `if (a > 0 && b > 0)`에서 a>0은 참이지만 b>0은 거짓인 경우를 테스트하지 않으면 잠재적 결함을 놓칠 수 있다. 이러한 구문 커버리지의 한계를 보완하기 위해 조건의 참/거짓을 모두 테스트하는 더 강력한 커버리지 기준은 무엇인가?

**정답**: 결정 커버리지
