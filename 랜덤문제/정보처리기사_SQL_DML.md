# 정보처리기사 SQL - DML 주관식 단답형 문제

> **정답 키워드:** DML (Data Manipulation Language)
> **문제 유형:** 주관식 단답형 (빈칸 채우기 + 직접 질문 + 비교 설명 혼합)

---

## 문제 1 (빈칸 채우기 유형)

**질문:** 데이터베이스 언어 중, 데이터를 삽입(INSERT), 조회(SELECT), 수정(UPDATE), 삭제(DELETE)하는 등 데이터 자체를 조작하는 명령어들의 집합을 **[     ]** 이라고 한다.

**정답:** DML (Data Manipulation Language)

---

## 문제 2 (직접 질문 유형)

**질문:** 데이터베이스에서 테이블의 데이터를 추가하거나, 기존 데이터를 변경·삭제·검색하는 등 데이터를 직접 조작하는 SQL 명령어(INSERT, SELECT, UPDATE, DELETE)들을 총칭하는 용어는 무엇인가?

**정답:** DML (Data Manipulation Language)

---

## 문제 3 (비교 설명 유형)

**질문:** 데이터베이스 언어 중 **DDL(Data Definition Language)**은 테이블이나 인덱스의 구조를 정의하거나 변경하는 CREATE, ALTER, DROP 등의 명령어를 다룬다. 반면, **[     ]** 은(는) 이미 정의된 테이블 내의 실제 데이터를 삽입·조회·수정·삭제하는 INSERT, SELECT, UPDATE, DELETE 등의 명령어를 다루는 데이터 조작 언어이다.

**정답:** DML (Data Manipulation Language)

---

## AI 프롬프트 작성 가이드

DML를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: DML (Data Manipulation Language)
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- DML의 결정적 키워드(INSERT, SELECT, UPDATE, DELETE, 데이터 조작, 데이터 조회, 데이터 삽입·수정·삭제)를 문제 설명에 배치하여 정답을 유도
- DDL과 DML의 차이점을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: DML (Data Manipulation Language)"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "INSERT, SELECT, UPDATE, DELETE" vs "CREATE, ALTER, DROP (DDL)", "데이터 조작" vs "구조 정의", "실제 데이터 다루기" vs "스키마 정의" 등 DML의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
