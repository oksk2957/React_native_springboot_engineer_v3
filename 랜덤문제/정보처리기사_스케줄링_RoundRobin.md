# 정보처리기사 스케줄링 - Round Robin

## AI 프롬프트 작성 가이드

- **정답 키워드 명시**: 프롬프트에 "정답: ( )"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
- **유형 지정**: 1.빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제만 랜덤형식의 질문으로 다양성이 보장되어 작성한다. 2.정답은 ()으로 고정된다.
- **결정적 구분자 제시**: ( )에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
- **해설 미포함**: 질문과 정답만 작성하며 해설은 작성하지 않는다.

---

## 문제 1

**유형**: 빈칸 채우기

일정한 시간 할당량(Time Quantum)을 정하여 각 프로세스에게 순환적으로 CPU를 할당하는 선점형 스케줄링 알고리즘을 ( )이라고 한다.

**정답**: Round Robin

---

## 문제 2

**유형**: 직접 질문

Round Robin 스케줄링에서 시간 할당량이 너무 크면 FCFS와 유사해지고, 너무 작으면 문맥 교환(Context Switching) 오버헤드가 증가한다. 이 알고리즘의 이름을 쓰시오.

**정답**: Round Robin

---

## 문제 3

**유형**: 비교 설명

모든 프로세스가 공정하게 CPU를 사용할 수 있도록 순환하며 CPU를 할당하는 선점형 스케줄링 알고리즘의 이름을 쓰시오.

**정답**: Round Robin
