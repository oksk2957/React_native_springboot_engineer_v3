## 문제 2 (직접 질문 유형)

**질문:** 프로세스 내에서 독립적인 실행 흐름을 가지며, 같은 프로세스의 메모리 공간을 공유하면서도 독자적인 스택과 레지스터를 보유하는 경량화된 실행 단위의 이름은 무엇인가?

**정답:** 스레드 (Thread)

## 문제 3 (비교 설명 유형)

**질문:** 운영체제에서 **프로세스(Process)**는 독립적인 메모리 공간(코드, 데이터, 힙, 스택)을 가지는 실행 단위로, 프로세스 간 통신(IPC)이 필요하며 문맥 교환 오버헤드가 크다. 반면, **[     ]** 은(는) 프로세스 내에서 코드·데이터·힙 영역을 공유하면서 독자적인 스택과 프로그램 카운터를 갖는 경량 실행 단위로, 문맥 교환이 빠르고 자원 공유가 용이하다.

**정답:** 스레드 (Thread)

## AI 프롬프트 작성 가이드

스레드를 정답으로 하는 주관식 단답형 문제를 AI에게 생성하도록 요청할 때, 다음 프롬프트 형식을 사용한다:

```
정보처리기사 실기 시험용 주관식 단답형 문제 3개를 생성하라.
- 정답: 스레드 (Thread)
- 유형: 빈칸 채우기와 직접 질문과 비교 설명을 혼합
- 스레드의 결정적 키워드(프로세스 내 실행 단위, 코드·데이터·힙 공유, 독자적 스택, 프로그램 카운터, 경량 실행 단위, 문맥 교환 빠름, 자원 공유 용이)를 문제 설명에 배치하여 정답을 유도
- 프로세스와 스레드의 차이점을 1개 문제에 포함
```

**프롬프트 작성 핵심 원칙:**
1. **정답 키워드 명시**: 프롬프트에 "정답: 스레드 (Thread)"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
2. **유형 지정**: 빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제 형식의 다양성이 보장된다.
3. **결정적 구분자 제시**: "프로세스 내 실행 단위" vs "독립 실행 단위(프로세스)", "메모리 공유" vs "독립 메모리 공간", "경량" vs "중량", "문맥 교환 빠름" vs "문맥 교환 느림" 등 스레드의 고유 키워드를 프롬프트에 포함하면, AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
