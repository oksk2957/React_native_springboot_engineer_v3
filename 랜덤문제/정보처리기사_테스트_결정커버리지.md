# 정보처리기사 테스트 - 결정 커버리지

## AI 프롬프트 작성 가이드

- **정답 키워드 명시**: 프롬프트에 "정답: ( )"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
- **유형 지정**: 1.빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제만 랜덤형식의 질문으로 다양성이 보장되어 작성한다. 2.정답은 ()으로 고정된다.
- **결정적 구분자 제시**: ( )에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
- **해설 미포함**: 질문과 정답만 작성하며 해설은 작성하지 않는다.

---

## 문제 1

**유형**: 빈칸 채우기

프로그램 내의 모든 결정점(Decision Point)에서 참(True)과 거짓(False)의 모든 가능한 결과를 최소 한 번 이상 수행하도록 테스트 케이스를 설계하는 커버리지 기준을 ( )이라고 한다.

**정답**: 결정 커버리지

---

## 문제 2

**유형**: 직접 질문

결정 커버리지는 모든 분기(Branch)를 최소 한 번씩 실행하도록 테스트 케이스를 설계하는 기법으로, 구문 커버리지보다 더 강력한 테스트 기준이다. 이 커버리지를 만족하면 구문 커버리지도 자동으로 만족되지만, 개별 조건의 모든 조합을 커버하지는 못한다. 결정 커버리지의 또 다른 이름은 무엇인가?

**정답**: 분기 커버리지

---

## 문제 3

**유형**: 비교 설명

구문 커버리지는 코드의 모든 라인을 실행하면 통과하지만, 조건 분기의 한쪽 경로(참 또는 거짓)만 실행되는 경우를 발견하지 못할 수 있다. 반면에 결정 커버리지는 모든 분기의 참과 거짓을 모두 실행하도록 요구한다. 예를 들어 `if (a > 0)`에서 a>0이 참인 경우만 테스트하면 구문 커버리지는 만족되지만, 결정 커버리지는 만족되지 않는다. 이러한 결정 커버리지의 목적은 무엇인가?

**정답**: 모든 논리적 분기 경로 검증
