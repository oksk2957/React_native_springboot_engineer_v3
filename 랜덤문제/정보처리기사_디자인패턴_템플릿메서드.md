## 템플릿 메서드(Template Method) 패턴 핵심 개념 정리

| 구성 요소 | 설명 |
|-----------|------|
| **추상 클래스(AbstractClass)** | 템플릿 메서드(알고리즘 골격)와 추상 메서드, 훅 메서드 선언 |
| **구체 클래스(ConcreteClass)** | 추상 메서드와 훅 메서드를 구현 |
| **템플릿 메서드** | 알고리즘의 실행 순서를 정의하는 메서드 (final로 선언 권장) |
| **추상 메서드** | 하위 클래스에서 반드시 구현해야 하는 메서드 |
| **훅(Hook) 메서드** | 선택적으로 재정의할 수 있는 메서드 (기본 구현 제공) |

### 주요 특징
- **알고리즘 골격 고정**: 상위 클래스에서 알고리즘의 전체 구조 정의
- **세부 구현 분리**: 하위 클래스에서 구체적인 단계 구현
- **코드 재사용**: 공통 알고리즘을 상위 클래스에서 한 번만 정의
- **확장성**: 하위 클래스에서 특정 단계만 재정의하여 확장 가능
- **대표 예시**: Java의 AbstractList, HttpServlet의 doGet/doPost, JUnit의 setUp/tearDown

## AI 프롬프트 작성 가이드 (랜덤 질문 생성용)

### 프롬프트 작성 핵심 원칙
1. **정답 고정**: 파일명의 마지막 부분(관련정답)이 모든 문제의 고정 정답이 되어야 함
2. **질문 랜덤화**: 같은 정답이라도 질문은 다양한 형태로 랜덤하게 생성
3. **맥락 다양화**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락에서 질문 생성

### 예시 프롬프트
```
정보처리기사 시험을 위한 주관식 단답형 문제를 생성해주세요.

[핵심 규칙]
- 고정 정답: "템플릿 메서드 패턴(Template Method Pattern)"
- 모든 문제의 정답은 반드시 "템플릿 메서드 패턴(Template Method Pattern)"이어야 함
- 질문은 다양한 형태로 랜덤하게 생성

[조건]
- 주제: 디자인 패턴 중 템플릿 메서드 패턴(Template Method Pattern)
- 문제 유형: 주관식 단답형 (빈칸 채우기)
- 난이도: 중급
- 문제 수: 3개
- 고정 정답: 템플릿 메서드 패턴(Template Method Pattern) (모든 문제 동일)

[질문 다양화 예시]
- 문제 1: 정의 관점 (알고리즘의 골격을 상위 클래스에서 정의하고 하위 클래스에서 세부 구현)
- 문제 2: 구조 관점 (메서드 실행 순서는 상위 클래스에서 고정, 구현은 하위 클래스에서 담당)
- 문제 3: 특징 관점 (추상 메서드와 훅 메서드를 활용하여 알고리즘 단계 커스터마이징)

[출력 형식]
각 문제는 다음 형식으로 작성:
- 질문: 다양한 표현으로 질문
- 정답: 템플릿 메서드 패턴(Template Method Pattern) (고정)
```

### 프롬프트 작성 시 주의사항
- **정답 고정 확인**: 모든 문제의 정답이 파일명의 관련정답과 일치하는지 검증
- **질문 중복 방지**: 같은 표현의 질문이 반복되지 않도록 다양화
- **맥락 다양성**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락 활용
- **난이도 조절**: 기출문제 수준의 난이도로 설정
- **오답 유도 방지**: 정답이 명확하고 유일하도록 문제 설계
