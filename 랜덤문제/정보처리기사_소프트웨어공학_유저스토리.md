# 정보처리기사 소프트웨어공학 - 유저스토리

## AI 프롬프트 작성 가이드

- **정답 키워드 명시**: 프롬프트에 "정답: ( )"을 직접 지정하면 AI가 정답을 고정하여 문제를 역설계한다.
- **유형 지정**: 1.빈칸 채우기 / 직접 질문 / 비교 설명 유형을 명시하면 문제만 랜덤형식의 질문으로 다양성이 보장되어 작성한다. 2.정답은 ()으로 고정된다.
- **결정적 구분자 제시**: ( )에 해당하는 고유 키워드를 프롬프트에 포함하고 AI가 오답과 정답을 명확히 구분하는 문제를 생성한다.
- **해설 미포함**: 질문과 정답만 작성하며 해설은 작성하지 않는다.

---

## 문제 1

**유형**: 빈칸 채우기

"~로서, ~를 하고 싶으니, ~가 필요하다"의 형태로 작성되며, 사용자의 관점에서 요구사항을 표현하는 기법을 ( )라고 한다.

**정답**: (유저스토리)

---

## 문제 2

**유형**: 직접 질문

애자일 방법론에서 유저스토리를 구현하기 위해 설정하는 짧은 개발 기간(보통 1~4주)을 무엇이라고 하는가?

**정답**: (스프린트)

---

## 문제 3

**유형**: 비교 설명

유저스토리의 INVEST 원칙에서 'I'는 Independent(독립적), 'N'은 Negotiable(협상 가능), 'V'는 Valuable(가치), 'E'는 Estimable(추정 가능), 'S'는 Small(작은 크기), 'T'는 Testable(테스트 가능)을 의미한다. 이 원칙은 ( )의 품질을 평가하는 기준으로 사용된다.

**정답**: (유저스토리)
