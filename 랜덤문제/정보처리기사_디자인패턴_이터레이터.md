## 이터레이터(Iterator) 패턴 핵심 개념 정리

| 구성 요소 | 설명 |
|-----------|------|
| **Iterator 인터페이스** | 순회에 필요한 메서드(next, hasNext, remove 등) 선언 |
| **구체 Iterator(ConcreteIterator)** | 실제 순회 로직을 구현하는 클래스 |
| **Aggregate 인터페이스** | Iterator를 생성하는 팩토리 메서드 선언 |
| **구체 Aggregate(ConcreteAggregate)** | 실제 컬렉션을 저장하고 Iterator를 생성하는 클래스 |

### 주요 특징
- **내부 구조 은닉**: 컬렉션의 내부 구조를 노출하지 않고 순회 가능
- **일관된 인터페이스**: 다양한 컬렉션에 대해 동일한 순회 인터페이스 제공
- **단일 책임 원칙**: 순회 로직을 컬렉션에서 분리하여 독립 관리
- **동시 순회**: 여러 Iterator가 동시에 동일한 컬렉션을 순회 가능
- **대표 예시**: Java의 Iterator, C++ STL의 iterator, Python의 __iter__/__next__
