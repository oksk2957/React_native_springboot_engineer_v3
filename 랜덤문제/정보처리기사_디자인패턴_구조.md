## 구조(Structural) 패턴 핵심 개념 정리

| 패턴 | 설명 |
|------|------|
| **어댑터(Adapter)** | 인터페이스가 호환되지 않는 클래스들을 함께 동작하도록 변환 |
| **브릿지(Bridge)** | 추상화와 구현을 분리하여 독립적으로 변형 가능 |
| **컴포지트(Composite)** | 개별 객체와 복합 객체를 동일하게 다루는 계층 구조 |
| **데코레이터(Decorator)** | 객체에 동적으로 새로운 기능을 추가 |
| **퍼사드(Facade)** | 복잡한 서브시스템에 대한 단순한 인터페이스 제공 |
| **플라이웨이트(Flyweight)** | 공유를 통해 메모리 사용 최소화 |
| **프록시(Proxy)** | 다른 객체에 대한 대리자(Proxy)를 제공하여 접근 제어 |

## AI 프롬프트 작성 가이드 (랜덤 질문 생성용)

### 프롬프트 작성 핵심 원칙
1. **정답 고정**: 파일명의 마지막 부분(관련정답)이 모든 문제의 고정 정답이 되어야 함
2. **질문 랜덤화**: 같은 정답이라도 질문은 다양한 형태로 랜덤하게 생성
3. **맥락 다양화**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락에서 질문 생성

### 예시 프롬프트
```
정보처리기사 시험을 위한 주관식 단답형 문제를 생성해주세요.

[핵심 규칙]
- 고정 정답: "구조 패턴(Structural Pattern)"
- 모든 문제의 정답은 반드시 "구조 패턴(Structural Pattern)"여야 함
- 질문은 다양한 형태로 랜덤하게 생성

[조건]
- 주제: 디자인 패턴 중 구조 패턴(Structural Pattern)
- 문제 유형: 주관식 단답형 (빈칸 채우기)
- 난이도: 중급
- 문제 수: 3개
- 고정 정답: 구조 패턴(Structural Pattern) (모든 문제 동일)

[질문 다양화 예시]
- 문제 1: 정의 관점 (클래스나 객체를 조합하여 더 큰 구조를 만드는 패턴)
- 문제 2: 특징 관점 (상속과 객체 합성을 활용하여 구조를 확장하는 패턴)
- 문제 3: 포함 관점 (어댑터, 브릿지, 컴포지트 등이 속하는 패턴 분류)

[출력 형식]
각 문제는 다음 형식으로 작성:
- 질문: 다양한 표현으로 질문
- 정답: 구조 패턴(Structural Pattern) (고정)
```

### 프롬프트 작성 시 주의사항
- **정답 고정 확인**: 모든 문제의 정답이 파일명의 관련정답과 일치하는지 검증
- **질문 중복 방지**: 같은 표현의 질문이 반복되지 않도록 다양화
- **맥락 다양성**: 정의, 특징, 유형, 관계, 예시 등 다양한 맥락 활용
- **난이도 조절**: 기출문제 수준의 난이도로 설정
- **오답 유도 방지**: 정답이 명확하고 유일하도록 문제 설계
