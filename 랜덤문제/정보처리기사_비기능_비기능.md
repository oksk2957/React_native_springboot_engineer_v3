# 정보처리기사 비기능(Non-functional) 주관식 단답형 문제

> **출제 범위**: 소프트웨어 공학 - 비기능 요구사항(Non-functional Requirements)
> **문제 유형**: 주관식 단답형
> **난이도**: 중급
> **고정 정답**: 비기능(Non-functional)

---

## 문제 1

### 질문
시스템이 어떻게 동작해야 하는지에 관한 요구사항으로, 성능, 보안, 가용성 등의 품질 속성을 정의하는 요구사항은 무엇인가?

### 정답
**비기능(Non-functional)**

### 해설
- 비기능 요구사항은 시스템이 어떻게 동작해야 하는지에 관한 요구사항입니다.
- 성능, 보안, 가용성, 확장성, 유지보수성 등의 품질 속성을 정의합니다.
- 기능 요구사항과는 달리 시스템의 품질 특성을 규정합니다.
- 예: "시스템은 99.9%의 가용성을 보장해야 한다", "응답 시간은 1초 이내여야 한다"

---

## 문제 2

### 질문
시스템의 성능, 보안성, 신뢰성 등 품질 속성을 규정하며, 시스템이 제공하는 서비스의 품질 수준을 정의하는 요구사항 유형은 무엇인가?

### 정답
**비기능(Non-functional)**

### 해설
- 비기능 요구사항은 시스템의 품질 속성을 규정하는 요구사항입니다.
- 성능(응답 시간, 처리량), 보안성(인증, 권한), 신뢰성(장애 복구, 가용성) 등을 포함합니다.
- 기능 요구사항이 "무엇을(What)" 하는지 정의한다면, 비기능 요구사항은 "얼마나 잘(How well)" 하는지 정의합니다.
- ISO/IEC 25010 소프트웨어 품질 모델과 밀접한 관련이 있습니다.

---

## 문제 3

### 질문
기능 요구사항과 대비되는 개념으로, 시스템의 품질 특성이나 제약 조건을 명시하는 요구사항은 무엇인가?

### 정답
**비기능(Non-functional)**

### 해설
- 비기능 요구사항은 기능 요구사항과 대비되는 개념입니다.
- 기능 요구사항이 시스템이 제공해야 하는 기능을 정의한다면, 비기능 요구사항은 그 기능의 품질 수준을 정의합니다.
- ISO/IEC 25010에서 정의한 소프트웨어 품질 특성(기능성, 신뢰성, 사용성, 효율성, 유지보수성, 이식성, 보안성)과 관련이 있습니다.
- 프로젝트 성공의 중요한 척도로 활용됩니다.

---

## 비기능 요구사항 분류

| 분류 | 설명 | 예시 |
|------|------|------|
| 성능 | 시스템 처리 능력 | 응답 시간, 처리량, 자원 사용량 |
| 보안 | 데이터 보호 수준 | 인증, 권한, 암호화, 감사 |
| 가용성 | 시스템 가동 시간 | 99.9% 가용성, 장애 복구 시간 |
| 확장성 | 성장 대응 능력 | 사용자 증가, 데이터 증가 대응 |
| 유지보수성 | 변경 용이성 | 모듈화, 문서화, 테스트 용이성 |
| 이식성 | 환경 이전 용이성 | 플랫폼 독립성, 표준 준수 |
| 사용성 | 사용자 편의성 | 직관성, 학습 용이성, 접근성 |
| 신뢰성 | 오류 없는 동작 | 결함 허용, 복구 능력, 정확성 |

---

## 기능 vs 비기능 요구사항 비교

| 구분 | 기능 요구사항 | 비기능 요구사항 |
|------|-------------|---------------|
| 초점 | 시스템이 무엇을 하는가 | 시스템이 어떻게 동작하는가 |
| 내용 | 기능, 서비스, 동작 | 품질, 제약, 조건 |
| 예시 | "사용자는 로그인할 수 있다" | "로그인 응답 시간은 1초 이내" |
| 측정 | 기능 완료 여부 | 품질 지표 달성 여부 |
| 중요도 | 필수 기능 | 품질 수준 |

## AI 프롬프트 작성 가이드 (랜덤 질문 생성용)

### 프롬프트 작성 핵심 원칙
1. **정답 고정**: 파일명의 마지막 부분(관련정답)이 모든 문제의 고정 정답이 되어야 함
2. **질문 랜덤화**: 같은 정답이라도 질문은 다양한 형태로 랜덤하게 생성
3. **맥락 다양화**: 정의, 특징, 유형, 관계, 비교 등 다양한 맥락에서 질문 생성

### 예시 프롬프트
```
정보처리기사 시험을 위한 주관식 단답형 문제를 생성해주세요.

[핵심 규칙]
- 고정 정답: "비기능(Non-functional)"
- 모든 문제의 정답은 반드시 "비기능(Non-functional)"여야 함
- 질문은 다양한 형태로 랜덤하게 생성

[조건]
- 주제: 소프트웨어 공학의 비기능 요구사항(Non-functional Requirements)
- 문제 유형: 주관식 단답형 (빈칸 채우기)
- 난이도: 중급
- 문제 수: 3개
- 고정 정답: 비기능(Non-functional) (모든 문제 동일)

[질문 다양화 예시]
- 문제 1: 정의 관점 (시스템 품질 속성 정의)
- 문제 2: 특징 관점 (성능, 보안, 가용성 등 품질 속성)
- 문제 3: 관계 관점 (기능 요구사항과의 대비)

[출력 형식]
각 문제는 다음 형식으로 작성:
- 질문: 다양한 표현으로 질문
- 정답: 비기능(Non-functional) (고정)
- 해설: 해당 질문의 맥락에 맞는 해설
```

### 프롬프트 작성 시 주의사항
- **정답 고정 확인**: 모든 문제의 정답이 파일명의 관련정답과 일치하는지 검증
- **질문 중복 방지**: 같은 표현의 질문이 반복되지 않도록 다양화
- **맥락 다양성**: 정의, 특징, 유형, 관계, 비교, 예시 등 다양한 맥락 활용
- **난이도 조절**: 기출문제 수준의 난이도로 설정
- **오답 유도 방지**: 정답이 명확하고 유일하도록 문제 설계
- **실무 연계**: 실제 프로젝트에서의 비기능 요구사항 예시 포함
