# 수정계획안13: 코드탭 3버그 수정

## 🎯 목표
코드탭(ProgrammingScreen)의 3가지 버그를 수정하여 정상 작동하도록 합니다.

## 🐛 버그 목록

### 버그 1: 오답노트 탭 누락
- **현재 상태**: 플래시카드, 주관식 2개 탭만 존재
- **기대 상태**: 플래시카드, 주관식, 오답노트 3개 탭
- **영향**: 사용자가 오답을 복습할 수 없음

### 버그 2: 플래시카드 필터링 실패
- **현재 상태**: `LANGUAGE_KEYWORDS`가 `card.category='프로그래밍언어'`에서 세부 언어(C, Java, Python)를 매칭하지 못함
- **근본 원인**: 서버에서 모든 카드의 category를 '프로그래밍언어'로 반환 → 클라이언트에서 C/Java/Python 구분 불가
- **기대 상태**: 선택한 언어(C/Java/Python)에 해당하는 카드만 표시
- **해결 방안**: 서버 SQL에서 `prog_language` 컬럼을 category로 반환하도록 수정

### 버그 3: 주관식 5지선다 미표시
- **현재 상태**: 주관식 탭에서 보기(option1~5)가 표시되지 않음
- **근본 원인**: `ProblemQueryMapper.xml`의 `selectProgrammingCardsByLanguage` 쿼리가 option1~5를 NULL로 하드코딩
- **기대 상태**: option1~5 데이터가 있으면 5지선다 형태로 표시
- **해결 방안**: SQL에서 option1~5 컬럼을 실제로 반환하도록 수정

## 🔧 수정 계획

### Step 1: SQL 쿼리 수정 (서버)
**파일**: `backend/src/main/resources/mapper/ProblemQueryMapper.xml`

```xml
<select id="selectProgrammingCardsByLanguage" resultType="map">
    SELECT
        id,
        question AS front_text,
        answer AS back_text,
        explanation,
        option1, option2, option3, option4, option5,
        prog_language AS category,
        difficulty,
        'SUBJECTIVE' AS card_type
    FROM programming_language_problems
    WHERE prog_language = #{language}
</select>
```

**변경 포인트**:
- `NULL AS option1~5` → `option1, option2, option3, option4, option5`
- `card_type`을 동적으로 판별 (option1 존재 여부)

### Step 2: 프론트엔드 필터링 로직 수정
**파일**: `InformationExamApp/src/screens/ProgrammingScreen.tsx`

**현재 코드** (200-250번 줄):
```typescript
const filteredCards = allCards.filter(card => {
    return LANGUAGE_KEYWORDS[currentLanguage].some(keyword =>
        card.frontText.includes(keyword) ||
        card.backText?.includes(keyword) ||
        card.category?.includes(keyword)
    );
});
```

**수정 방향**:
- 서버에서 이미 `prog_language`로 필터링하므로 클라이언트 필터링 제거
- 또는 `card.category === currentLanguage`로 단순 비교

### Step 3: 오답노트 탭 추가

- 플래시카드/주관식과 동일한 UI 패턴
