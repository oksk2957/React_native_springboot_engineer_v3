   ```typescript
   function fisherYatesShuffle<T>(array: T[]): T[] {
     const result = [...array];
     for (let i = result.length - 1; i > 0; i--) {
       const j = Math.floor(Math.random() * (i + 1));
       [result[i], result[j]] = [result[j], result[i]];
     }
     return result;
   }
   ```

**핵심 발견**:
- Memory는 "잔존 1~4건"이라고 하지만, 실제 코드는 11건 잔존
- 수정계획안12가 "완료"로 표시되었지만, 실제로는 #81~#83 미완료
- "코드 수정"과 "기능 완료"는 다름

**교훈**:
- 런타임 검증 없이 "완료" 표시 금지
- Memory 갱신 시 코드 상태 재확인 필수
- Explore agent로 전체 스캔 정기적 실시

**우선순위 결정**:
- Phase 1 (토큰체인)이 가장 중요: 이게 없으면 통계/오답/랭킹 전체 무효
- Phase 2 (랭킹 NULL)는 Phase 1과 병렬 가능하지만 순차 권장
- Phase 5 (Git 커밋)은 모든 작업 완료 후 실행

**Memory v61 예상**: 잔존 3건 (Google 400, JWT Refresh, E1)
