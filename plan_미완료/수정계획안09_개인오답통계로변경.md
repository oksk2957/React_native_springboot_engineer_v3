# 수정계획안09: 개인 오답 통계로 변경 (전체 랭킹 → 개인별 틀린 문제 통계)

## 1. 현황 분석

### 1.1 현재 구현 (v43 완료)
- **통계 화면**: 전체 사용자별 오답 카운트 랭킹
- **데이터 소스**: `wrong_answer_bookmark` 테이블 (사용자별 총 오답 수)
- **표시 형식**: TOP3 메달 + 4~100위 순차 표시
- **문제점**: 여러 사용자의 랭킹이지만, 실제로는 개인 1명의 통계가 필요함

### 1.2 요구사항
- **변경**: 개인 1명이 푼 오답 문제별 통계
- **형식 유지**: 1~3위 메달, 4~30위 순차, 31위+ 10개씩 페이지네이션
- **데이터**: 개인이 가장 많이 틀린 문제 순으로 정렬

## 2. 수정 범위

### 2.1 백엔드 (Backend)
#### (1) Mapper XML 변경
**파일**: `backend/src/main/resources/mapper/MypageStatisticsMapper.xml`

**현재 쿼리** (lines 48-58):
```xml
<select id="getWrongAnswerRanking" resultType="com.example.informationexam.dto.statistics.WrongAnswerRankingRow">
    SELECT
        u.id as userId,
        u.username,
        u.nickname,
        COUNT(wab.id) as wrongAnswerCount
    FROM wrong_answer_bookmark wab
    JOIN users u ON wab.user_id = u.id
    WHERE wab.marked_at >= NOW() - INTERVAL '7 days'
    GROUP BY u.id, u.username, u.nickname
    ORDER BY wrongAnswerCount DESC
    LIMIT 100
</select>
```

**변경 쿼리** (개인 오답 문제별 통계):
```xml
<select id="getWrongAnswerRanking" resultType="com.example.informationexam.dto.statistics.WrongAnswerRankingRow">
    SELECT
        qp.id as problemId,
        qp.subject,
        qp.question_text,
        COUNT(wab.id) as wrongAnswerCount
    FROM wrong_answer_bookmark wab
    JOIN question_problems qp ON wab.problem_id = qp.id
    WHERE wab.user_id = #{userId}
      AND wab.marked_at >= NOW() - INTERVAL '7 days'
    GROUP BY qp.id, qp.subject, qp.question_text
    ORDER BY wrongAnswerCount DESC
    LIMIT 100
</select>
```

**변경 포인트**:
- `JOIN users u` → `JOIN question_problems qp`
- `u.id, u.username, u.nickname` → `qp.id, qp.subject, qp.question_text`
- `WHERE` 조건에 `wab.user_id = #{userId}` 추가 (개인 필터)
