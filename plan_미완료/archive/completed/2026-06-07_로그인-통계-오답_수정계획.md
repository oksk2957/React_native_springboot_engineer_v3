## 2. 문제별 원인 분석

### 2.1 문제 1: 로그인 400 에러

**증상**: `/api/auth/google` POST 시 400 Bad Request

**원인 분석**:
1. `GoogleAuthController.java:36-50`에서 `idToken`이 null이거나 비어있으면 400 반환
2. `UserService.loginWithSupabase()`에서 `SupabaseTokenVerifierService.getEmail()` 호출 시 예외 발생 → `IllegalArgumentException` → 400
3. 로그에서 "백엔드 교환 실패" 메시지가 나오므로 **JWT 검증 단계에서 실패**하고 있음

**가능성**:
- Supabase JWT가 만료되었거나 유효하지 않음
- JWKS 엔드포인트(`/auth/v1/.well-known/jwks.json`)에서 공개키를 가져오는 데 실패
- 토큰의 `email` claim이 없거나 `sub`도 없음

**확인 필요**:
- `backend/server.log`에서 `[AUTH]` 태그가 포함된 로그 확인
- Supabase JWT가 실제로 전달되고 있는지 확인

---

### 2.2 문제 2: 통계 랭킹 빈값

**증상**: "과목별 시도 횟수 랭킹"에 데이터가 없음

**원인 분석**:
1. `MypageStatisticsMapper.xml:79-110`의 `selectSubjectRanking` 쿼리
2. 쿼리가 `user_answer` 테이블을 기반으로 작성됨
3. 하지만 `AnswerService.submitAnswer()`에서는 `StudySessionItem`도 함께 저장하고 있음 (라인 101-134)
4. **쿼리가 `study_session_item` 테이블을 사용하지 않음**
5. `StatisticsService.getSubjectRanking()`이 `selectSubjectRanking`을 호출하므로 데이터가 비어 있음

**해결 방향**:
- 옵션 A: `selectSubjectRanking` 쿼리를 `study_session_item` 기반으로 변경
- 옵션 B: `user_answer` 테이블에 데이터가 실제로 있는지 확인 후 쿼리 수정

**우선순위**: 옵션 B (데이터 확인) → 옵션 A (쿼리 수정)

---

### 2.3 문제 3: 오답탭 문제 미표시

**증상**: 오답 탭에 문제 리스트가 표시되지 않음

**원인 분석**:
1. `UserAnswerApiController.buildWrongAnswerResponse()` (라인 109-141)
2. `userAnswerRepository.findByUserIdAndIsCorrect(userId, false)`로 오답 조회
3. `problemRepository.findAllById(referenceIds)`로 **Problem 엔티티만 조회**
4. **문제**: `SUBJECTIVE`, `PROGRAMMING_LANGUAGE` 타입은 `problem` 테이블이 아닌 다른 테이블 사용
   - `SUBJECTIVE` → `subjective_problems` 테이블
   - `PROGRAMMING_LANGUAGE` → `programming_language_problems` 테이블
5. 따라서 해당 타입의 문제는 `problemMap.get()`에서 null → "문제를 찾을 수 없습니다"

**해결 방향**:
- `MypageStatisticsMapper.xml`의 `selectWrongAnswers` 쿼리처럼 UNION ALL로 모든 문제 타입 처리
- 또는 `UserAnswerApiController`에서 타입별로ProblemRepository 분기 처리

**우선순위**: Mapper XML 쿼리 재사용이 더 간단함

## 3. 수정 계획

### 3.1 문제 1: 로그인 400 에러 수정

**단계**:

**파일**:

- `backend/server.log`

### 3.2 문제 2: 통계 랭킹 빈값 수정

**단계**:
1. `user_answer` 테이블에 실제 데이터가 있는지 확인
2. `study_session_item` 테이블에도 데이터가 있는지 확인
3. `selectSubjectRanking` 쿼리를 `study_session_item` 기반으로 변경
4. `StatisticsService`에서 새로운 쿼리 호출하도록 수정

**파일**:
- `backend/src/main/resources/mapper/MypageStatisticsMapper.xml`
- `backend/src/main/java/com/example/informationexam/mapper/MypageStatisticsMapper.java`
- `backend/src/main/java/com/example/informationexam/service/StatisticsService.java`

---

### 3.3 문제 3: 오답탭 문제 미표시 수정

**단계**:
1. `UserAnswerApiController.buildWrongAnswerResponse()` 수정
2. `MypageStatisticsMapper.selectWrongAnswers()` 쿼리 재사용
3. 또는 UNION ALL로 모든 문제 타입 처리하는 새로운 쿼리 작성

**파일**:
