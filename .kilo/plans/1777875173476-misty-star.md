# Plan: Reset Opencode Model to Default

## Status
DRAFT - Awaiting validation

## Overview
Reset the currently connected opencode model to the default model by modifying configuration.

## Requirements
- Reset opencode model to default
- Remove current custom model settings

## Technical Decisions
- Opencode uses `opencode.json` for config (not `kilo.json`)
- Global config: `~/.config/opencode/opencode.json`
- Project config: `opencode.json` in project root
- Project config overrides global config
- Model setting is in `model` field

## Implementation Steps

### Phase 1: Locate and Modify Config
1. Check if `opencode.json` exists in project root
2. If exists, remove or reset the `model` field
3. If not exists, create `opencode.json` with default model settings
4. Alternatively, modify global config at `~/.config/opencode/opencode.json`

### Phase 2: Alternative Locations
1. Check `~/.config/opencode/opencode.json` for global settings
2. Both project and global config can set the model

## Config Format
```json
{
  "$schema": "https://opencode.ai/config.json",
  "model": "anthropic/claude-sonnet-4-5"
}
```

To reset to default, either:
- Remove the `model` field entirely (Opencode will use its default)
- Or set to a known default model
- Location: Project `opencode.json` or global `~/.config/opencode/opencode.json`

## Verification
- Run `opencode` and verify default model is shown
- Check model selection in TUI
